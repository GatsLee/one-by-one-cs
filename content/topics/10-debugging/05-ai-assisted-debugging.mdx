## 핵심 개념

### 1. AI 디버깅의 현실

AI(ChatGPT, Claude, Copilot 등)는 디버깅의 강력한 도구이지만, **만능이 아니다**. AI를 잘 쓰려면 AI가 잘하는 것과 못하는 것을 알아야 한다.

```
AI가 잘하는 것:

  ✓ 에러 메시지 해석 ("이 에러가 무슨 뜻이야?")
  ✓ 일반적인 버그 패턴 인식 (off-by-one, null 참조 등)
  ✓ 라이브러리 사용법 안내 ("이 함수 어떻게 쓰는 거야?")
  ✓ 코드 리뷰 (문법 오류, 비효율 지적)
  ✓ 보일러플레이트 코드 생성

AI가 못하는 것:

  ✗ 실제 런타임 환경 접근 (내 서버 상태, DB 데이터)
  ✗ 프로젝트 전체 맥락 이해 (수백 개 파일의 관계)
  ✗ 비결정적 버그 재현 (가끔 발생하는 문제)
  ✗ 비즈니스 로직 검증 ("이 계산이 맞나?")
  ✗ 최신 라이브러리 API (학습 데이터 이후 변경)
```

### 2. AI에게 좋은 질문하는 법

AI의 답변 품질은 **질문의 품질**에 비례한다. 좋은 질문의 3요소: 에러 메시지 + 관련 코드 + 시도한 것.

```
나쁜 질문:
  "코드가 안 돼요. 도와주세요."
  → AI: "더 자세한 정보를 주세요..." (쓸모없는 답변)

보통 질문:
  "TypeError가 나요."
  "TypeError: Cannot read properties of undefined (reading 'name')"
  → AI: "undefined에 접근하고 있네요" (맞지만 뻔한 답)

좋은 질문:
  "React 컴포넌트에서 API를 호출한 뒤 user.name을 렌더링하는데
   TypeError: Cannot read properties of undefined (reading 'name')
   에러가 납니다.

   ```jsx
   const [user, setUser] = useState();
   useEffect(() => {
     fetch('/api/user').then(r => r.json()).then(setUser);
   }, []);
   return <p>{user.name}</p>;
   ```

   useEffect가 비동기라서 첫 렌더링 시 user가 undefined인 것 같은데,
   옵셔널 체이닝 외에 더 좋은 패턴이 있나요?"
  → AI: 구체적이고 실행 가능한 해결책 제시
```

```
좋은 AI 디버깅 질문 템플릿:

  1. 상황: "~를 하고 있는데"
  2. 에러: "~라는 에러가 발생합니다"
  3. 코드: (관련 코드 첨부)
  4. 시도: "~를 해봤지만 안 됩니다"
  5. 환경: Python 3.11, Next.js 15 등
```

### 3. AI 답변 검증하기

AI는 자신감 있게 **틀린 답**을 줄 수 있다. 이를 "환각(hallucination)"이라 한다. 반드시 검증해야 한다.

```
AI 답변 검증 체크리스트:

  □ 제안한 코드가 실제로 실행되는가?
  □ 사용한 함수/메서드가 실제로 존재하는가?
  □ 라이브러리 버전이 내 환경과 맞는가?
  □ 부작용(side effect)은 없는가?
  □ 에러를 해결하되 새로운 버그를 만들지 않는가?
```

```
AI 환각이 자주 발생하는 경우:

  1. 존재하지 않는 API/메서드를 추천
     → 공식 문서에서 확인

  2. 오래된 문법/패턴 제안
     → "이 코드가 [라이브러리] [버전]에서 동작하나요?" 확인

  3. 비슷하지만 다른 라이브러리의 코드 혼합
     → import 경로가 올바른지 확인

  4. 해결은 되지만 비효율적인 코드
     → "이 방법의 시간 복잡도는?" 물어보기
```

### 4. AI 디버깅 워크플로우

```
버그 발생!
    │
    ▼
1단계: 스스로 에러 메시지 읽기 (30초)
    │ "에러 타입은? 위치는?"
    │
    ├─ 바로 해결됨 → 끝
    │
    ▼
2단계: 직접 디버깅 시도 (5분)
    │ print/디버거로 변수 확인
    │ 관련 코드 검토
    │
    ├─ 해결됨 → 끝
    │
    ▼
3단계: AI에게 질문 (좋은 질문 형식으로)
    │ 에러 + 코드 + 시도한 것 포함
    │
    ▼
4단계: AI 답변 검증
    │ 코드 실행, 공식 문서 확인
    │
    ├─ 해결됨 → 끝
    │
    ▼
5단계: AI에게 후속 질문
    │ "시도했는데 다른 에러가 나요: ..."
    │ 새로운 에러 메시지와 코드 첨부
    │
    ├─ 해결됨 → 끝
    │
    ▼
6단계: 검색/커뮤니티 (Stack Overflow, GitHub Issues)
    │ AI가 해결 못하는 환경 특정 문제일 수 있음
```

### 5. 멘탈 모델 구축 — AI에 의존하지 않는 힘

AI가 답을 줘도 **"왜" 그게 해결책인지** 이해해야 같은 문제를 다시 겪지 않는다.

```
나쁜 습관:
  에러 발생 → AI에게 복붙 → AI 답변 복붙 → 작동하면 끝
  → 같은 유형의 버그를 매번 AI에게 물어봐야 함

좋은 습관:
  에러 발생 → AI에게 질문 → 답변을 이해
  → "왜 이 해결책이 작동하는 거야?"
  → "이 패턴이 발생하는 근본 원인은?"
  → 같은 유형의 버그를 혼자 해결할 수 있게 됨
```

```
AI에게 추가로 물어볼 것:

  "해결됐는데, 왜 이렇게 되는 거야?"
  "이 에러가 발생하는 근본 원인은 뭐야?"
  "이런 종류의 버그를 예방하려면 어떻게 코딩해야 해?"
  "관련 CS 개념이 있어?"
```

### 6. AI 코드 생성 후 디버깅

AI가 생성한 코드에서 버그가 나는 건 자연스러운 일이다. AI 코드를 검증하는 체크리스트:

```
AI 생성 코드 검증 체크리스트:

  □ 경계값 테스트: 빈 배열, null, 0, 빈 문자열로 테스트
  □ 에러 처리: try/catch가 적절한가? 에러를 삼키지 않는가?
  □ 타입 확인: TypeScript면 타입이 맞는가?
  □ 보안: 사용자 입력을 신뢰하고 있지 않은가?
  □ 성능: 큰 데이터에서도 동작하는가? (N이 10만이면?)
  □ 부작용: 전역 변수를 건드리지 않는가?
```
## 상황
Next.js 16 App Router에서 SSE(Server-Sent Events) 스트리밍을 구현 중입니다.

## 에러
클라이언트에서 EventSource로 연결하면 첫 번째 이벤트만 받고 연결이 끊깁니다.

## 코드
서버 (app/api/stream/route.ts):
```ts
export async function GET() {
  const stream = new ReadableStream({
    start(controller) {
      controller.enqueue("data: hello\n\n");
      controller.enqueue("data: world\n\n");
      controller.close();
    },
  });
  return new Response(stream, {
    headers: { "Content-Type": "text/event-stream" },
  });
}
```

클라이언트:
```ts
const es = new EventSource("/api/stream");
es.onmessage = (e) => console.log(e.data);
```

## 시도한 것
- Content-Type 헤더 확인 (text/event-stream 맞음)
- controller.close()를 제거하면 연결은 유지되지만 이벤트가 안 옴

## 환경
Next.js 16, Node.js 22, Chrome 130
```


## 연습 문제

### Q1. AI에게 디버깅을 요청할 때 반드시 포함해야 할 3가지는?

<details>
<summary>정답 보기</summary>
(1) 에러 메시지 전문, (2) 관련 코드, (3) 이미 시도한 것. 추가로 환경(언어 버전, 프레임워크)을 명시하면 더 정확한 답을 얻을 수 있다.
</details>

### Q2. AI가 제안한 코드를 검증하지 않고 그대로 사용하면 어떤 문제가 생길 수 있나?

<details>
<summary>정답 보기</summary>
(1) 존재하지 않는 API 사용으로 새 에러 발생, (2) 보안 취약점 포함, (3) 비효율적 코드로 성능 저하, (4) 원래 문제는 해결되지만 새로운 버그 도입, (5) 학습 기회 상실로 같은 문제 반복.
</details>

### Q3. "AI에게 복붙하고 AI 답변을 복붙하는" 패턴의 문제는?

<details>
<summary>정답 보기</summary>
근본 원인을 이해하지 못해 같은 유형의 버그를 매번 AI에게 물어봐야 한다. 멘탈 모델이 형성되지 않아 개발 실력이 정체된다. AI가 환각으로 잘못된 답을 줄 때 판별할 수 없다.
</details>
