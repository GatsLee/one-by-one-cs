## 핵심 개념

### 1. 에러 메시지는 지도다

프로그램이 멈추면 에러 메시지가 출력된다. 많은 입문자가 빨간 글씨를 보면 당황하지만, 에러 메시지는 **"여기가 문제야"** 라고 알려주는 지도다. 읽는 법만 알면 디버깅 시간을 90% 줄일 수 있다.

```
에러 메시지를 읽는 3단계:

1. 에러 타입 확인   →  "무슨 종류의 문제인가?"
2. 에러 내용 읽기   →  "구체적으로 뭐가 잘못됐나?"
3. 위치(스택 트레이스) →  "코드 어디에서 발생했나?"
```

### 2. 스택 트레이스(Stack Trace) 읽기

스택 트레이스는 에러가 발생하기까지 **함수 호출 경로**를 보여준다. 가장 중요한 건 **가장 아래(마지막) 줄**이다.

#### Python 스택 트레이스

```python
# 에러를 일으키는 코드
def calculate_average(numbers):
    total = sum(numbers)
    return total / len(numbers)

def process_data(data):
    scores = data["scores"]    # 여기서 KeyError 발생 가능
    return calculate_average(scores)

result = process_data({"name": "Kim"})  # "scores" 키 없음
```

```
Traceback (most recent call last):         ← "역추적" 시작
  File "main.py", line 9, in <module>      ← 호출 시작점
    result = process_data({"name": "Kim"})
  File "main.py", line 6, in process_data  ← 중간 호출
    scores = data["scores"]
KeyError: 'scores'                          ← ★ 에러 타입 + 원인
```

```
읽는 순서 (아래에서 위로):

   ① KeyError: 'scores'           →  딕셔너리에 'scores' 키가 없음
   ② line 6, in process_data      →  process_data 함수, 6번째 줄
   ③ line 9, in <module>          →  9번째 줄에서 호출됨

   → 해결: data에 "scores" 키가 있는지 먼저 확인
```

#### JavaScript 스택 트레이스

```javascript
// 에러를 일으키는 코드
function getUser(id) {
  const user = users.find(u => u.id === id);
  return user.name;  // user가 undefined일 때 에러
}
```

```
TypeError: Cannot read properties of undefined (reading 'name')
    at getUser (app.js:3:15)         ← ★ 에러 발생 위치
    at handleRequest (server.js:42:10)
    at processRequest (server.js:15:5)
```

```
읽는 순서 (위에서 아래로 — Python과 반대!):

   ① TypeError                    →  타입 에러 (undefined에서 속성 읽기 시도)
   ② at getUser (app.js:3:15)     →  app.js 3번째 줄, 15번째 글자
   ③ 'name'을 읽으려 했다         →  user가 undefined

   → 해결: user가 존재하는지 먼저 확인
```

```
★ 핵심 차이:
  Python  → 스택 트레이스를 아래에서 위로 읽는다 (마지막이 에러)
  JS/TS   → 스택 트레이스를 위에서 아래로 읽는다 (첫 줄이 에러)
```

### 3. 자주 만나는 에러 타입

#### Python 에러 타입

```
┌─────────────────┬──────────────────────────────────────────┐
│ 에러 타입        │ 의미                                      │
├─────────────────┼──────────────────────────────────────────┤
│ SyntaxError     │ 문법 오류 (괄호 안 닫음, 콜론 빠짐)         │
│ NameError       │ 정의되지 않은 변수 사용                      │
│ TypeError       │ 잘못된 타입 연산 (문자열 + 숫자 등)          │
│ IndexError      │ 리스트 범위 밖 인덱스 접근                   │
│ KeyError        │ 딕셔너리에 없는 키 접근                     │
│ AttributeError  │ 객체에 없는 속성/메서드 호출                 │
│ ImportError     │ 모듈을 찾을 수 없음                         │
│ ValueError      │ 값이 예상 범위 밖 (int("abc") 등)           │
│ FileNotFoundError│ 파일 경로가 잘못됨                         │
└─────────────────┴──────────────────────────────────────────┘
```

#### JavaScript/TypeScript 에러 타입

```
┌──────────────────┬──────────────────────────────────────────┐
│ 에러 타입         │ 의미                                      │
├──────────────────┼──────────────────────────────────────────┤
│ SyntaxError      │ 문법 오류 (JSON.parse 실패 등)              │
│ ReferenceError   │ 정의되지 않은 변수 참조                      │
│ TypeError        │ undefined/null에서 속성 접근, 함수 아닌 것 호출 │
│ RangeError       │ 배열 크기 초과, 스택 오버플로                │
│ URIError         │ URI 인코딩/디코딩 실패                      │
└──────────────────┴──────────────────────────────────────────┘
```

### 4. HTTP 에러 코드

웹 개발에서 자주 만나는 에러 코드:

```
┌──────┬────────────────┬──────────────────────────────────┐
│ 코드  │ 이름            │ 의미와 대응                        │
├──────┼────────────────┼──────────────────────────────────┤
│ 400  │ Bad Request    │ 요청 데이터가 잘못됨 → 파라미터 확인  │
│ 401  │ Unauthorized   │ 인증 필요 → 로그인/토큰 확인         │
│ 403  │ Forbidden      │ 권한 없음 → 권한 설정 확인           │
│ 404  │ Not Found      │ URL이 잘못됨 → 경로 확인            │
│ 405  │ Method Not     │ GET/POST 등 메서드가 잘못됨          │
│      │ Allowed        │                                  │
│ 500  │ Internal Error │ 서버 코드 에러 → 서버 로그 확인       │
│ 502  │ Bad Gateway    │ 프록시/게이트웨이 문제               │
│ 503  │ Service        │ 서버 과부하/점검 중                  │
│      │ Unavailable    │                                  │
└──────┴────────────────┴──────────────────────────────────┘
```

### 5. 에러 메시지로 검색하는 법

에러를 직접 해결할 수 없을 때, 효과적으로 검색하는 방법:

```
검색 공식:

  [언어/프레임워크] + [에러 타입] + [핵심 메시지]

예시:
  ✗ "에러가 났어요"
  ✗ "python error line 42"

  ✓ "python KeyError dictionary"
  ✓ "javascript TypeError cannot read property of undefined"
  ✓ "react useEffect cleanup function"
  ✓ "next.js 500 internal server error api route"
```

```
검색 팁:

1. 에러 메시지에서 내 변수명/파일명은 제거
   ✗ "KeyError: 'my_special_scores_variable'"
   ✓ "python KeyError dictionary key not found"

2. 특정 값보다 패턴으로 검색
   ✗ "TypeError at line 42 of main.py"
   ✓ "python TypeError NoneType has no attribute"

3. Stack Overflow가 뜨면 질문보다 "답변"을 먼저 읽기

4. 최신 답변 우선 (3년 이상 된 답변은 라이브러리 버전이 다를 수 있음)
```
