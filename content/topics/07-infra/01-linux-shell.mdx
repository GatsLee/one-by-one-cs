## 핵심 개념

### 1. Linux 파일시스템 구조

리눅스는 **단일 루트(`/`) 트리 구조**를 사용한다. Windows처럼 C:, D: 드라이브가 없고
모든 것이 `/`에서 시작하는 하나의 트리에 마운트된다.

```
/                          ← 루트 디렉터리 (모든 것의 시작)
├── home/                  ← 사용자 홈 디렉터리
│   ├── gatslee/           ← 개별 사용자 공간
│   │   ├── Desktop/
│   │   ├── Documents/
│   │   └── .bashrc        ← 숨김 파일 (dot file)
│   └── another_user/
├── etc/                   ← 시스템 설정 파일
│   ├── nginx/nginx.conf
│   ├── hosts
│   ├── passwd
│   └── ssh/sshd_config
├── var/                   ← 가변 데이터 (로그, 캐시)
│   ├── log/               ← 시스템 로그
│   │   ├── syslog
│   │   └── nginx/
│   ├── cache/
│   └── lib/               ← 데이터베이스 파일 등
├── usr/                   ← 사용자 프로그램
│   ├── bin/               ← 일반 명령어 (ls, grep 등)
│   ├── lib/               ← 라이브러리
│   ├── local/             ← 직접 설치한 프로그램
│   └── share/             ← 공유 데이터
├── tmp/                   ← 임시 파일 (재부팅 시 삭제)
├── bin/                   ← 필수 명령어 (최근 /usr/bin 심볼릭 링크)
├── sbin/                  ← 시스템 관리 명령어
├── dev/                   ← 디바이스 파일
├── proc/                  ← 프로세스/커널 정보 (가상 파일시스템)
├── sys/                   ← 하드웨어 정보 (가상 파일시스템)
├── opt/                   ← 서드파티 소프트웨어
├── boot/                  ← 부트로더, 커널 이미지
├── mnt/                   ← 임시 마운트 포인트
└── media/                 ← 이동식 미디어 자동 마운트
```

**FHS (Filesystem Hierarchy Standard)**: 리눅스 배포판들이 따르는 디렉터리 구조 표준이다.
이 표준 덕분에 Ubuntu든 CentOS든 기본 구조가 동일하다.

핵심 구분:
- **`/etc`**: 설정(Configuration) → "etcetera"의 약자
- **`/var`**: 변하는 데이터(Variable) → 로그, 메일, DB
- **`/usr`**: 읽기 전용 사용자 데이터 → 설치된 프로그램
- **`/tmp`**: 누구나 쓸 수 있는 임시 공간 → 재부팅 시 삭제

### 2. 파일/디렉터리 명령어

#### 기본 탐색과 조작

```bash
# 현재 위치 확인
pwd                        # /home/gatslee

# 디렉터리 이동
cd /var/log                # 절대경로 이동
cd ~                       # 홈 디렉터리로 이동
cd ..                      # 상위 디렉터리로 이동
cd -                       # 이전 디렉터리로 이동

# 파일/디렉터리 목록
ls                         # 기본 목록
ls -la                     # 숨김 파일 포함 + 상세 정보
ls -lh                     # 사람이 읽기 쉬운 크기 표시
ls -lt                     # 수정 시간 순 정렬

# 디렉터리 생성
mkdir myproject            # 단일 디렉터리
mkdir -p a/b/c/d           # 중간 디렉터리까지 모두 생성

# 복사/이동/삭제
cp file1.txt file2.txt     # 파일 복사
cp -r dir1/ dir2/          # 디렉터리 재귀 복사
mv old.txt new.txt         # 이름 변경 / 이동
rm file.txt                # 파일 삭제
rm -rf directory/          # 디렉터리 강제 재귀 삭제 (주의!)
```

#### 파일 권한 (Permission)

```
파일 권한 읽는 법:

  -rwxr-xr--  1  gatslee  dev  4096  Jan 15 10:30  script.sh
  │├─┤├─┤├─┤     │        │    │     │              │
  │ │   │  │     소유자   그룹  크기   수정일         파일명
  │ │   │  └─ Others: r-- (읽기만)
  │ │   └──── Group:  r-x (읽기+실행)
  │ └──────── Owner:  rwx (읽기+쓰기+실행)
  └────────── 파일 타입: - (일반파일), d (디렉터리), l (심볼릭 링크)

권한 비트:
  r (read)    = 4
  w (write)   = 2
  x (execute) = 1

예시:
  rwx = 4+2+1 = 7
  r-x = 4+0+1 = 5
  r-- = 4+0+0 = 4
  → chmod 754 script.sh
```

```bash
# 권한 변경
chmod 755 script.sh        # rwxr-xr-x
chmod +x script.sh         # 실행 권한 추가 (심볼릭)
chmod u+w,g-w file.txt     # 소유자 쓰기 추가, 그룹 쓰기 제거

# 소유자 변경
chown gatslee:dev file.txt # 소유자:그룹 변경
chown -R gatslee:dev dir/  # 재귀적으로 변경
```

### 3. 텍스트 처리

리눅스의 철학: **"하나의 일을 잘 하는 작은 프로그램을 파이프로 조합한다."**

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  cat     │───▶│  grep    │───▶│  awk     │───▶│  sort    │
│ 파일 읽기 │ |  │ 패턴 필터│ |  │ 필드 추출│ |  │ 정렬     │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
     stdout ──▶ stdin     stdout ──▶ stdin     stdout ──▶ stdin
```

#### 기본 명령어

```bash
# cat: 파일 내용 출력
cat /etc/hosts                  # 전체 내용
cat -n script.sh                # 줄번호 포함

# head / tail: 처음/끝 부분 출력
head -20 access.log             # 처음 20줄
tail -f /var/log/syslog         # 실시간 로그 추적 (follow)

# grep: 패턴 검색
grep "ERROR" app.log            # ERROR 포함 줄 찾기
grep -i "error" app.log         # 대소문자 무시
grep -r "TODO" ./src/           # 재귀 검색
grep -n "def " *.py             # 줄번호 포함
grep -v "DEBUG" app.log         # 패턴 제외 (invert)
grep -c "ERROR" app.log         # 매칭 줄 수 카운트
grep -E "ERROR|WARN" app.log    # 확장 정규식 (OR)

# sed: 스트림 편집 (Stream Editor)
sed 's/old/new/' file.txt       # 첫 번째 매칭만 치환
sed 's/old/new/g' file.txt      # 전체 치환 (global)
sed -i 's/old/new/g' file.txt   # 파일 직접 수정 (in-place)
sed -n '10,20p' file.txt        # 10~20번째 줄만 출력

# awk: 필드 기반 텍스트 처리
awk '{print $1}' file.txt       # 첫 번째 필드
awk -F: '{print $1}' /etc/passwd # 구분자를 :로 지정
awk '$3 > 100 {print $1, $3}' data.txt  # 조건부 출력
```

#### 파이프(|)와 리다이렉션(>, >>)

```
파이프 (Pipe):  프로그램 A의 출력을 프로그램 B의 입력으로 연결
리다이렉션:     출력을 파일로 보냄

┌─────────────────────────────────────────────────┐
│  파이프:       cmd1 | cmd2 | cmd3               │
│                stdout → stdin → stdout → stdin  │
│                                                 │
│  리다이렉션:   cmd > file     (덮어쓰기)         │
│                cmd >> file    (추가)              │
│                cmd 2> error   (에러만 파일로)     │
│                cmd &> all     (출력+에러 모두)    │
│                cmd < input    (파일을 stdin으로)  │
└─────────────────────────────────────────────────┘
```

```bash
# 파이프 활용 예시
ps aux | grep python              # 파이썬 프로세스 찾기
cat access.log | sort | uniq -c | sort -rn | head -10
#               정렬   중복제거+카운트  역순정렬   상위10개

# 리다이렉션 예시
echo "hello" > output.txt         # 덮어쓰기
echo "world" >> output.txt        # 추가
ls /nonexistent 2> error.log      # 에러를 파일로
ls /home /nonexistent &> all.log  # 모두 파일로
```

### 4. 프로세스 관리

```
┌──────────────────────────────────────────────────────┐
│                    프로세스 상태                       │
│                                                      │
│   생성 ──▶ 실행(Running) ──▶ 종료(Terminated)         │
│              │       ▲                                │
│              ▼       │                                │
│           대기(Sleeping/Waiting)                       │
│              │       ▲                                │
│              ▼       │                                │
│           정지(Stopped)  ← Ctrl+Z                     │
│                                                      │
│   포그라운드(fg) ◀──▶ 백그라운드(bg)                   │
└──────────────────────────────────────────────────────┘
```

```bash
# 프로세스 확인
ps aux                       # 모든 프로세스 (BSD 스타일)
ps -ef                       # 모든 프로세스 (System V 스타일)
top                          # 실시간 모니터링 (q로 종료)
htop                         # 개선된 top (설치 필요)

# 프로세스 종료
kill <PID>                   # SIGTERM (정상 종료 요청, 시그널 15)
kill -9 <PID>                # SIGKILL (강제 종료, 시그널 9)
kill -HUP <PID>              # SIGHUP (설정 다시 읽기, 시그널 1)
killall python               # 이름으로 모든 프로세스 종료
pkill -f "python app.py"     # 패턴 매칭으로 종료

# 백그라운드 실행
./long_task.sh &             # 백그라운드에서 시작
Ctrl+Z                       # 현재 작업 정지 (suspend)
bg                            # 정지된 작업을 백그라운드로
fg                            # 백그라운드 작업을 포그라운드로
jobs                          # 현재 셸의 작업 목록

# 셸 종료 후에도 실행 유지
nohup ./server.sh &           # 세션 종료 후에도 유지
disown %1                     # 이미 실행 중인 작업을 분리

# 서비스 관리 (systemd)
systemctl status nginx         # 서비스 상태 확인
systemctl start nginx          # 시작
systemctl stop nginx           # 중지
systemctl restart nginx        # 재시작
systemctl enable nginx         # 부팅 시 자동 시작
systemctl disable nginx        # 자동 시작 해제
journalctl -u nginx -f         # 서비스 로그 실시간 확인
```

**주요 시그널 정리:**

| 시그널 | 번호 | 설명 |
|--------|------|------|
| SIGHUP | 1 | 터미널 종료 / 설정 다시 읽기 |
| SIGINT | 2 | 인터럽트 (Ctrl+C) |
| SIGKILL | 9 | 강제 종료 (잡을 수 없음) |
| SIGTERM | 15 | 정상 종료 요청 (기본값) |
| SIGSTOP | 19 | 정지 (잡을 수 없음) |
| SIGTSTP | 20 | 터미널 정지 (Ctrl+Z) |

### 5. 셸 스크립팅 기초

셸 스크립트는 반복적인 작업을 자동화하는 핵심 도구다.

#### 기본 구조

```bash
#!/bin/bash
# ↑ shebang: 이 스크립트를 해석할 인터프리터 지정

# 변수 (= 양쪽에 공백 없음!)
NAME="gatslee"
COUNT=42
TODAY=$(date +%Y-%m-%d)    # 명령어 치환

echo "안녕하세요, $NAME"
echo "오늘: ${TODAY}"
echo "인자 개수: $#"       # 스크립트에 전달된 인자 수
echo "첫 번째 인자: $1"    # 위치 매개변수
echo "모든 인자: $@"
echo "종료 코드: $?"       # 이전 명령의 종료 코드 (0=성공)
```

#### 조건문 (if)

```bash
#!/bin/bash

FILE="/etc/hosts"

# 파일 존재 확인
if [ -f "$FILE" ]; then
    echo "$FILE 존재합니다"
elif [ -d "$FILE" ]; then
    echo "$FILE 은 디렉터리입니다"
else
    echo "$FILE 이 없습니다"
fi

# 숫자 비교
NUM=10
if [ $NUM -gt 5 ]; then      # -gt: greater than
    echo "5보다 큽니다"
fi

# 문자열 비교
if [ "$NAME" = "gatslee" ]; then
    echo "이름 일치"
fi

# 조건 연산자 정리:
# -f 파일이 존재하고 일반 파일인가
# -d 디렉터리가 존재하는가
# -e 파일/디렉터리가 존재하는가
# -z 문자열이 비어 있는가
# -n 문자열이 비어있지 않은가
# -eq, -ne, -gt, -lt, -ge, -le : 숫자 비교
```

#### 반복문 (for / while)

```bash
#!/bin/bash

# for 루프: 리스트 순회
for fruit in apple banana cherry; do
    echo "과일: $fruit"
done

# for 루프: 파일 순회
for file in /var/log/*.log; do
    echo "로그 파일: $file"
    wc -l "$file"
done

# for 루프: C 스타일
for ((i=1; i<=5; i++)); do
    echo "숫자: $i"
done

# while 루프
COUNTER=0
while [ $COUNTER -lt 5 ]; do
    echo "카운터: $COUNTER"
    COUNTER=$((COUNTER + 1))
done

# while 루프: 파일 줄 단위 읽기
while IFS= read -r line; do
    echo "줄: $line"
done < input.txt
```

#### 함수

```bash
#!/bin/bash

# 함수 정의
greet() {
    local name=$1       # local 변수 (함수 내부에서만)
    local time=$2
    echo "안녕하세요, ${name}님! 지금은 ${time}입니다."
    return 0            # 종료 코드 반환 (0=성공)
}

# 함수 호출
greet "gatslee" "오후 3시"

# 반환값 활용
check_file() {
    if [ -f "$1" ]; then
        return 0        # 성공
    else
        return 1        # 실패
    fi
}

if check_file "/etc/hosts"; then
    echo "파일이 존재합니다"
fi
```

### 6. SSH (Secure Shell)

```
SSH 연결 흐름:

┌──────────┐         인터넷          ┌──────────────┐
│  클라이언트 │ ◀──── 암호화 터널 ────▶ │  서버         │
│  (내 PC)   │    TCP 포트 22         │  (원격 서버)  │
└──────────┘                         └──────────────┘

인증 방식:
1. 비밀번호 인증 — 간단하지만 취약
2. 키 기반 인증 — 공개키/개인키 쌍 사용 (권장)

  ┌─────────────────────────────────────────┐
  │  키 기반 인증 과정                       │
  │                                         │
  │  클라이언트                  서버          │
  │  ┌────────────┐         ┌──────────┐    │
  │  │ 개인키      │         │ 공개키    │    │
  │  │ (~/.ssh/    │  ──▶    │ ~/.ssh/  │    │
  │  │  id_ed25519)│  인증    │ authorized│   │
  │  └────────────┘         │ _keys    │    │
  │                         └──────────┘    │
  └─────────────────────────────────────────┘
```

```bash
# SSH 키 생성 (Ed25519 권장)
ssh-keygen -t ed25519 -C "gatslee@example.com"
# → ~/.ssh/id_ed25519      (개인키 — 절대 공유 금지!)
# → ~/.ssh/id_ed25519.pub  (공개키 — 서버에 등록)

# 공개키를 서버에 복사
ssh-copy-id user@server.com

# 원격 접속
ssh user@server.com
ssh -p 2222 user@server.com  # 포트 지정

# 파일 전송 (SCP)
scp local.txt user@server:/remote/path/
scp -r local_dir/ user@server:/remote/path/
scp user@server:/remote/file.txt ./local/

# 포트 포워딩 (로컬)
ssh -L 8080:localhost:3000 user@server
# → 내 PC의 localhost:8080 → 서버의 localhost:3000

# 포트 포워딩 (리모트)
ssh -R 9090:localhost:8080 user@server
# → 서버의 localhost:9090 → 내 PC의 localhost:8080
```

#### SSH Config 파일

```bash
# ~/.ssh/config
Host dev-server
    HostName 192.168.1.100
    User gatslee
    Port 2222
    IdentityFile ~/.ssh/id_ed25519

Host prod
    HostName production.example.com
    User deploy
    IdentityFile ~/.ssh/deploy_key
    ForwardAgent yes

Host jump
    HostName bastion.example.com
    User admin

Host internal
    HostName 10.0.0.50
    User gatslee
    ProxyJump jump         # jump 서버를 경유해서 접속
```

```bash
# config 설정 후 간단하게 접속
ssh dev-server             # ssh -p 2222 gatslee@192.168.1.100 과 동일
ssh internal               # bastion 경유 자동 접속
```

### 7. 환경변수

```
환경변수의 범위:

  ┌─────────────────────────────────────┐
  │  시스템 전체 (/etc/environment)      │
  │  ┌─────────────────────────────────┐│
  │  │  사용자 전체 (~/.bashrc)         ││
  │  │  ┌─────────────────────────────┐││
  │  │  │  현재 셸 세션 (export)       │││
  │  │  │  ┌─────────────────────────┐│││
  │  │  │  │  현재 명령 (VAR=x cmd)  ││││
  │  │  │  └─────────────────────────┘│││
  │  │  └─────────────────────────────┘││
  │  └─────────────────────────────────┘│
  └─────────────────────────────────────┘
```

```bash
printenv PATH              # 특정 환경변수 출력

# 환경변수 설정
export MY_VAR="hello"      # 현재 셸 + 자식 프로세스에서 유효
MY_VAR="hello"             # 현재 셸에서만 유효 (export 없으면)
VAR=value command           # 해당 명령에서만 유효

# PATH 추가
export PATH="$HOME/.local/bin:$PATH"   # 앞에 추가 (우선순위 높음)
export PATH="$PATH:/opt/custom/bin"    # 뒤에 추가

# 영구 설정 — ~/.bashrc 또는 ~/.zshrc에 추가
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc           # 변경사항 즉시 적용
```
