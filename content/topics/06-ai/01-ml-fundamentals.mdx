## 핵심 개념

### 1. 머신러닝이란?

머신러닝은 **데이터로부터 패턴을 학습하여 새로운 데이터에 대한 예측을 수행하는 알고리즘**이다.

### 2. 지도학습 vs 비지도학습

```
지도학습 (Supervised Learning):
  ┌────────────────────────────────────────────┐
  │  입력(X)          →  정답(y) = 라벨        │
  │  [집 크기, 방 수]  →  가격 3억             │  ← 회귀
  │  [이메일 텍스트]    →  스팸/정상            │  ← 분류
  │  "정답지가 있는 학습"                       │
  └────────────────────────────────────────────┘

비지도학습 (Unsupervised Learning):
  ┌────────────────────────────────────────────┐
  │  입력(X)만 있음    →  정답(y) 없음!        │
  │  스스로 패턴/구조를 발견                     │
  │  - 군집화: 비슷한 데이터를 그룹으로 묶기     │
  │  - 차원 축소: 핵심 특성만 추출              │
  └────────────────────────────────────────────┘
```

| 구분 | 지도학습 | 비지도학습 |
|------|---------|-----------|
| 라벨 | 있음 | 없음 |
| 목표 | 입력 → 출력 매핑 학습 | 데이터의 구조/패턴 발견 |
| 대표 알고리즘 | 선형회귀, 로지스틱회귀, 결정트리 | K-means, PCA |
| 적용 예시 | 가격 예측, 스팸 분류 | 고객 세분화, 이상 탐지 |

### 3. 선형 회귀 (Linear Regression)

입력 변수(X)와 출력 변수(y) 사이의 **선형 관계를 모델링**하는 가장 기본적인 알고리즘이다.

```
수식:  y = wx + b

  w: 가중치(weight) — 기울기
  b: 편향(bias) — y절편
  목표: 데이터에 가장 잘 맞는 직선을 찾는 것

비용 함수 — MSE (Mean Squared Error):
  MSE = (1/n) × Σ(yᵢ - ŷᵢ)²
  → MSE가 작을수록 예측이 정확함
```

### 4. 경사 하강법 (Gradient Descent)

비용 함수의 기울기(미분)를 따라 **점진적으로 최솟값을 찾아가는 최적화 알고리즘**이다.

```
비유: 안개 낀 산에서 골짜기 찾기
  - 현재 위치에서 가장 가파른 내리막 방향을 감지
  - 그 방향으로 한 걸음 이동 (학습률)

  업데이트 규칙:
  w = w - α × (∂J/∂w)

  α: 학습률 (learning rate)
     - 너무 크면: 최솟값을 건너뛰고 발산
     - 너무 작으면: 수렴이 매우 느림
```

### 5. 로지스틱 회귀 (Logistic Regression)

이름은 "회귀"이지만, 실제로는 **이진 분류(Binary Classification)** 알고리즘이다.

```
시그모이드(Sigmoid) 함수로 출력을 0~1로 변환:
  σ(z) = 1 / (1 + e^(-z))

  1.0 │              ──────
      │           ╱
  0.5 │─────────╱──────────
      │       ╱
  0.0 │──────
      └─────────────────── z

  출력 > 0.5 → 클래스 1 (양성)
  출력 < 0.5 → 클래스 0 (음성)

비용 함수: Binary Cross-Entropy
  J = -(1/n) × Σ[yᵢ × log(ŷᵢ) + (1-yᵢ) × log(1-ŷᵢ)]
```

### 6. 과적합 / 과소적합

```
과소적합              적절한 적합              과적합
(Underfitting)        (Good Fit)            (Overfitting)

  y │  *     *        y │  *     *          y │  *──╮  *
    │    ────────        │    ╱╲ *              │ ╱   ╲╱╲
    │  *  *              │  *╱  ╲              │╱ *    ╲ *
    └──────── x         └──────── x          └──────── x

  너무 단순한 모델      데이터의 진짜 패턴      노이즈까지 학습
  훈련/테스트 모두 나쁨   훈련/테스트 모두 좋음   훈련 좋지만 테스트 나쁨
```

| 구분 | 과소적합 | 과적합 |
|------|---------|--------|
| 원인 | 모델이 너무 단순 | 모델이 너무 복잡 |
| 해결 | 모델 복잡도 증가 | 정규화, 데이터 증가, 드롭아웃 |

### 7. 편향-분산 트레이드오프 (Bias-Variance Tradeoff)

```
총 오차 = 편향² + 분산 + 노이즈(줄일 수 없음)

  편향(Bias): 모델의 가정이 잘못되어 생기는 오차
  분산(Variance): 훈련 데이터에 따라 예측이 크게 변하는 정도

  모델 단순화 → 편향 ↑ 분산 ↓
  모델 복잡화 → 편향 ↓ 분산 ↑
```

### 8. 평가 지표

#### 혼동 행렬 (Confusion Matrix)

```
                  예측 양성(P)   예측 음성(N)
  실제 양성(P)  │    TP        │    FN       │
  실제 음성(N)  │    FP        │    TN       │

정확도 (Accuracy) = (TP + TN) / 전체
  ⚠ 클래스 불균형 시 오해의 소지

정밀도 (Precision) = TP / (TP + FP)
  → "양성 예측 중 진짜 양성" (FP 줄이기)

재현율 (Recall) = TP / (TP + FN)
  → "실제 양성 중 맞춘 비율" (FN 줄이기)

F1 = 2 × (Precision × Recall) / (Precision + Recall)

AUC-ROC: 임계값 전 범위의 분류 성능 (1.0=완벽, 0.5=랜덤)
```

### 9. 교차 검증 (K-fold Cross Validation)

```
5-Fold Cross Validation:

Fold 1: [검증, 훈련, 훈련, 훈련, 훈련] → 성능 A
Fold 2: [훈련, 검증, 훈련, 훈련, 훈련] → 성능 B
...
최종 성능 = (A + B + C + D + E) / 5

→ 모든 데이터가 한 번씩 검증에 사용, 신뢰성 있는 평가
```
