## 핵심 개념

### 배열 (Array)

배열은 **같은 타입의 데이터를 연속된 메모리 공간에 저장**하는 자료구조다. 아파트를 떠올려보자. 101호, 102호, 103호... 각 호실은 같은 크기이고, 번호만 알면 바로 찾아갈 수 있다.

```
메모리 주소:  1000   1004   1008   1012   1016
            +------+------+------+------+------+
배열:       |  10  |  20  |  30  |  40  |  50  |
            +------+------+------+------+------+
인덱스:       [0]    [1]    [2]    [3]    [4]
```

**핵심 특성:**
- **연속된 메모리**: 원소들이 메모리에 빈틈없이 나란히 저장된다
- **인덱스 접근**: `arr[3]`처럼 인덱스로 즉시 접근 가능 (O(1))
- **고정 크기**: 전통적인 배열은 생성 시 크기가 정해진다 (C 언어의 `int arr[5]`)

**인덱스 접근이 O(1)인 이유:**
```
원소의 주소 = 배열 시작 주소 + (인덱스 × 원소 크기)

예) arr[3]의 주소 = 1000 + (3 × 4바이트) = 1012
→ 계산 한 번으로 바로 찾아감!
```

### 연결 리스트 (Linked List)

연결 리스트는 **각 노드가 데이터와 다음 노드의 주소(포인터)를 가진** 자료구조다. 보물찾기 게임을 떠올려보자. 첫 번째 쪽지에 "다음 힌트는 나무 아래"라고 적혀 있고, 나무 아래 쪽지에 "다음 힌트는 벤치 옆"이라고 적혀 있는 식이다.

```
단일 연결 리스트 (Singly Linked List):

 head
  │
  v
+------+---+    +------+---+    +------+---+    +------+------+
|  10  | ──┼──> |  20  | ──┼──> |  30  | ──┼──> |  40  | None |
+------+---+    +------+---+    +------+---+    +------+------+
 [data|next]     [data|next]     [data|next]     [data|next]
```

**핵심 특성:**
- **비연속 메모리**: 노드들이 메모리 여기저기에 흩어져 있다
- **순차 접근**: n번째 원소를 찾으려면 처음부터 n번 따라가야 한다 (O(n))
- **유연한 크기**: 노드를 추가/삭제하기 쉽다

### 이중 연결 리스트 (Doubly Linked List)

이중 연결 리스트는 각 노드가 **이전 노드와 다음 노드 모두의 주소**를 가진다. 양방향 이동이 가능하다.

```
이중 연결 리스트 (Doubly Linked List):

        head                                              tail
         │                                                 │
         v                                                 v
  +----+------+---+    +---+------+---+    +---+------+----+
  |None|  10  | ──┼──> | ──┼  20  | ──┼──> | ──┼  30  |None|
  +----+------+---+    +---+------+---+    +---+------+----+
  [prev|data |next]    [prev|data |next]   [prev|data |next]
                  <──┤        <──┤
```

### Python의 list: 동적 배열

Python의 `list`는 전통적인 고정 크기 배열이 아니라 **동적 배열(Dynamic Array)**이다. 내부적으로는 C 배열을 사용하되, 공간이 부족하면 자동으로 더 큰 배열을 할당한다.

```
동적 배열의 확장 과정:

1) 초기 상태 (용량 4, 원소 4개 - 꽉 참)
   +---+---+---+---+
   | 1 | 2 | 3 | 4 |
   +---+---+---+---+

2) append(5) 호출 → 공간 부족!

3) 더 큰 배열(용량 8) 할당 후 복사
   +---+---+---+---+---+---+---+---+
   | 1 | 2 | 3 | 4 | 5 |   |   |   |
   +---+---+---+---+---+---+---+---+

   → 기존 배열은 메모리에서 해제
```

**Python list의 용량 증가 패턴:**
- 대략 현재 크기의 약 1.125배 + 상수만큼 증가 (정확한 공식은 CPython 버전마다 다름)
- `append()`는 **분할 상환 분석(Amortized Analysis)**으로 평균 O(1)

### 시간 복잡도 비교

```
+------------------+----------+---------------+-----------------+
|     연산         |   배열   | 단일 연결리스트 | 이중 연결리스트  |
+------------------+----------+---------------+-----------------+
| 인덱스 접근      |   O(1)   |     O(n)      |      O(n)       |
| 맨 앞 삽입       |   O(n)   |     O(1)      |      O(1)       |
| 맨 뒤 삽입       |  *O(1)   |     O(n)      |      O(1)       |
| 중간 삽입        |   O(n)   |  O(1)+탐색    |   O(1)+탐색     |
| 맨 앞 삭제       |   O(n)   |     O(1)      |      O(1)       |
| 맨 뒤 삭제       |   O(1)   |     O(n)      |      O(1)       |
| 중간 삭제        |   O(n)   |  O(1)+탐색    |   O(1)+탐색     |
| 검색 (비정렬)    |   O(n)   |     O(n)      |      O(n)       |
+------------------+----------+---------------+-----------------+
* 동적 배열의 경우 분할 상환 O(1)
```

### 언제 무엇을 선택하는가?

| 상황 | 추천 자료구조 | 이유 |
|------|-------------|------|
| 인덱스로 빠르게 접근해야 할 때 | 배열 (Python list) | O(1) 접근 |
| 데이터 크기를 미리 알고 변경이 적을 때 | 배열 | 메모리 효율적 |
| 앞쪽에서 삽입/삭제가 빈번할 때 | 연결 리스트 (deque) | O(1) 삽입/삭제 |
| 중간 삽입/삭제가 빈번할 때 | 연결 리스트 | 원소 이동 불필요 |
| 메모리를 절약해야 할 때 | 배열 | 포인터 오버헤드 없음 |
| 데이터 크기가 예측 불가할 때 | 연결 리스트 | 동적 확장 용이 |

**실무 팁:** Python에서는 대부분 `list`로 충분하다. 앞쪽 삽입/삭제가 빈번하면 `collections.deque`를 사용한다. 직접 연결 리스트를 구현하는 경우는 드물지만, **원리를 이해하는 것이 면접과 알고리즘 문제 풀이에 필수**다.

## 코드 예시

### 1. Python list의 동적 배열 동작 확인

```python
import sys

# Python list의 메모리 크기 변화를 관찰
nums = []
prev_size = 0

print(f"{'원소 수':>8} | {'메모리(bytes)':>13} | {'변화'}")
print("-" * 45)

for i in range(20):
    nums.append(i)
    current_size = sys.getsizeof(nums)
    change = "← 확장!" if current_size != prev_size else ""
    print(f"{len(nums):>8} | {current_size:>13} | {change}")
    prev_size = current_size

# 출력 예시 (CPython 3.12):
#   원소 수 |  메모리(bytes) | 변화
# ---------------------------------------------
#        1 |            88 | ← 확장!
#        2 |            88 |
#        3 |            88 |
#        4 |            88 |
#        5 |           120 | ← 확장!
#       ...
# → 용량이 꽉 찰 때마다 더 큰 배열을 할당함을 확인
```

### 2. 배열 연산 시간 측정

```python
import time

def measure_time(func, *args):
    """함수 실행 시간을 측정하는 헬퍼"""
    start = time.perf_counter()
    func(*args)
    return time.perf_counter() - start

n = 100_000

# 맨 뒤에 추가: O(1) 분할상환
lst = []
t = measure_time(lambda: [lst.append(i) for i in range(n)])
print(f"맨 뒤 추가 {n}회: {t:.4f}초")

# 맨 앞에 추가: O(n) — 매번 모든 원소를 한 칸씩 뒤로 밀어야 함
lst2 = []
t = measure_time(lambda: [lst2.insert(0, i) for i in range(n)])
print(f"맨 앞 추가 {n}회: {t:.4f}초")

# → 맨 앞 추가가 훨씬 느린 것을 확인할 수 있다
```

### 3. 단일 연결 리스트 구현

```python
class Node:
    """연결 리스트의 노드"""
    def __init__(self, data):
        self.data = data  # 저장할 데이터
        self.next = None  # 다음 노드를 가리키는 포인터

    def __repr__(self):
        return f"Node({self.data})"


class SinglyLinkedList:
    """단일 연결 리스트"""
    def __init__(self):
        self.head = None  # 리스트의 첫 번째 노드
        self._size = 0    # 원소 개수 추적

    def __len__(self):
        return self._size

    def is_empty(self):
        return self.head is None

    # --- 삽입 연산 ---

    def prepend(self, data):
        """맨 앞에 삽입 - O(1)"""
        new_node = Node(data)
        new_node.next = self.head  # 새 노드가 기존 head를 가리킴
        self.head = new_node       # head를 새 노드로 변경
        self._size += 1

    def append(self, data):
        """맨 뒤에 삽입 - O(n) (tail을 탐색해야 하므로)"""
        new_node = Node(data)
        if self.is_empty():
            self.head = new_node
        else:
            # 마지막 노드까지 이동
            current = self.head
            while current.next is not None:
                current = current.next
            current.next = new_node
        self._size += 1

    def insert_after(self, target_data, data):
        """특정 값 뒤에 삽입 - O(n) 탐색 + O(1) 삽입"""
        current = self.head
        while current is not None:
            if current.data == target_data:
                new_node = Node(data)
                new_node.next = current.next  # 새 노드가 다음 노드를 가리킴
                current.next = new_node       # 현재 노드가 새 노드를 가리킴
                self._size += 1
                return True
            current = current.next
        return False  # target_data를 찾지 못함

    # --- 삭제 연산 ---

    def delete_first(self):
        """맨 앞 삭제 - O(1)"""
        if self.is_empty():
            raise IndexError("빈 리스트에서 삭제할 수 없습니다")
        data = self.head.data
        self.head = self.head.next  # head를 다음 노드로 이동
        self._size -= 1
        return data

    def delete(self, data):
        """특정 값 삭제 - O(n)"""
        if self.is_empty():
            raise ValueError(f"{data}을(를) 찾을 수 없습니다")

        # 삭제할 노드가 head인 경우
        if self.head.data == data:
            self.head = self.head.next
            self._size -= 1
            return

        # head가 아닌 경우: 이전 노드를 추적하며 탐색
        prev = self.head
        current = self.head.next
        while current is not None:
            if current.data == data:
                prev.next = current.next  # 이전 노드가 다음 노드를 가리킴
                self._size -= 1
                return
            prev = current
            current = current.next

        raise ValueError(f"{data}을(를) 찾을 수 없습니다")

    # --- 탐색 연산 ---

    def search(self, data):
        """값 검색 - O(n)"""
        current = self.head
        index = 0
        while current is not None:
            if current.data == data:
                return index
            current = current.next
            index += 1
        return -1  # 찾지 못함

    # --- 유틸리티 ---

    def to_list(self):
        """연결 리스트를 Python list로 변환"""
        result = []
        current = self.head
        while current is not None:
            result.append(current.data)
            current = current.next
        return result

    def __repr__(self):
        if self.is_empty():
            return "빈 리스트"
        parts = []
        current = self.head
        while current is not None:
            parts.append(str(current.data))
            current = current.next
        return " -> ".join(parts) + " -> None"


# === 사용 예시 ===
ll = SinglyLinkedList()
ll.append(10)
ll.append(20)
ll.append(30)
print(ll)  # 10 -> 20 -> 30 -> None

ll.prepend(5)
print(ll)  # 5 -> 10 -> 20 -> 30 -> None

ll.insert_after(20, 25)
print(ll)  # 5 -> 10 -> 20 -> 25 -> 30 -> None

ll.delete(20)
print(ll)  # 5 -> 10 -> 25 -> 30 -> None

print(f"검색 25: 인덱스 {ll.search(25)}")  # 검색 25: 인덱스 2
print(f"길이: {len(ll)}")  # 길이: 4
```

### 4. 이중 연결 리스트 구현

```python
class DNode:
    """이중 연결 리스트의 노드"""
    def __init__(self, data):
        self.data = data
        self.prev = None  # 이전 노드 포인터
        self.next = None  # 다음 노드 포인터


class DoublyLinkedList:
    """이중 연결 리스트"""
    def __init__(self):
        self.head = None
        self.tail = None  # 마지막 노드 추적 → 뒤쪽 연산 O(1)
        self._size = 0

    def __len__(self):
        return self._size

    def append(self, data):
        """맨 뒤에 추가 - O(1) (tail 포인터 덕분)"""
        new_node = DNode(data)
        if self.head is None:
            self.head = self.tail = new_node
        else:
            new_node.prev = self.tail  # 새 노드의 prev = 기존 tail
            self.tail.next = new_node  # 기존 tail의 next = 새 노드
            self.tail = new_node       # tail 갱신
        self._size += 1

    def prepend(self, data):
        """맨 앞에 추가 - O(1)"""
        new_node = DNode(data)
        if self.head is None:
            self.head = self.tail = new_node
        else:
            new_node.next = self.head
            self.head.prev = new_node
            self.head = new_node
        self._size += 1

    def delete_last(self):
        """맨 뒤 삭제 - O(1) (단일 연결리스트는 O(n)이었음!)"""
        if self.tail is None:
            raise IndexError("빈 리스트")
        data = self.tail.data
        if self.head == self.tail:
            self.head = self.tail = None
        else:
            self.tail = self.tail.prev
            self.tail.next = None
        self._size -= 1
        return data

    def __repr__(self):
        parts = []
        current = self.head
        while current:
            parts.append(str(current.data))
            current = current.next
        return " <-> ".join(parts) if parts else "빈 리스트"


# === 사용 예시 ===
dll = DoublyLinkedList()
dll.append(10)
dll.append(20)
dll.append(30)
dll.prepend(5)
print(dll)  # 5 <-> 10 <-> 20 <-> 30

deleted = dll.delete_last()
print(f"삭제된 값: {deleted}")  # 삭제된 값: 30
print(dll)  # 5 <-> 10 <-> 20
```

### 5. collections.deque — 실무에서의 연결 리스트 대안

```python
from collections import deque

# deque는 내부적으로 이중 연결 리스트와 유사한 구조
# 양쪽 끝 삽입/삭제가 모두 O(1)

dq = deque([1, 2, 3])

dq.append(4)       # 오른쪽 추가:  [1, 2, 3, 4]
dq.appendleft(0)   # 왼쪽 추가:   [0, 1, 2, 3, 4]
dq.pop()           # 오른쪽 삭제:  [0, 1, 2, 3]
dq.popleft()       # 왼쪽 삭제:   [1, 2, 3]

print(list(dq))  # [1, 2, 3]

# list와 deque 성능 비교
import time

n = 100_000

# list: 앞쪽 삽입 O(n)
lst = []
start = time.perf_counter()
for i in range(n):
    lst.insert(0, i)
list_time = time.perf_counter() - start

# deque: 앞쪽 삽입 O(1)
dq = deque()
start = time.perf_counter()
for i in range(n):
    dq.appendleft(i)
deque_time = time.perf_counter() - start

print(f"list  맨 앞 삽입: {list_time:.4f}초")
print(f"deque 맨 앞 삽입: {deque_time:.4f}초")
# → deque가 수십~수백 배 빠름
```

## 정리

| 항목 | 배열 (Python list) | 단일 연결 리스트 | 이중 연결 리스트 |
|------|-------------------|----------------|----------------|
| 메모리 배치 | 연속 | 비연속 | 비연속 |
| 인덱스 접근 | O(1) | O(n) | O(n) |
| 맨 앞 삽입/삭제 | O(n) | O(1) | O(1) |
| 맨 뒤 삽입 | 분할상환 O(1) | O(n) (tail 없을 때) | O(1) |
| 맨 뒤 삭제 | O(1) | O(n) | O(1) |
| 메모리 오버헤드 | 낮음 | 포인터 1개/노드 | 포인터 2개/노드 |
| Python 구현 | list | 직접 구현 | collections.deque |
| 캐시 효율 | 높음 (연속 메모리) | 낮음 (흩어진 메모리) | 낮음 |
| 대표 사용처 | 랜덤 접근, 정렬 | 스택, 단방향 순회 | 큐, 양방향 순회 |
