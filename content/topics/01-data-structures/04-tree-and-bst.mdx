## 핵심 개념

### 트리(Tree)란?

트리는 **계층적 구조**를 가진 비선형 자료구조다. 실생활에서 가계도, 조직도, 파일 시스템 디렉토리 구조가 모두 트리 형태다.

```
        CEO (루트)
       /    \
     CTO    CFO
    / | \     \
  PM  PM  PM  회계팀장
 / \
개발자 개발자
```

### 트리 용어 정리

| 용어 | 설명 |
|------|------|
| **루트(Root)** | 트리의 최상위 노드 (부모가 없는 노드) |
| **리프(Leaf)** | 자식이 없는 노드 (말단 노드) |
| **간선(Edge)** | 노드와 노드를 잇는 연결선 |
| **깊이(Depth)** | 루트에서 해당 노드까지의 간선 수 |
| **높이(Height)** | 해당 노드에서 가장 먼 리프까지의 간선 수 |
| **차수(Degree)** | 노드의 자식 수 |
| **서브트리(Subtree)** | 특정 노드를 루트로 하는 부분 트리 |
| **레벨(Level)** | 같은 깊이에 있는 노드들의 집합 (깊이 + 1) |

```
레벨 1:        A          ← 루트, 깊이 0, 높이 3
              / \
레벨 2:      B   C        ← 깊이 1
            / \   \
레벨 3:    D   E   F      ← 깊이 2
          /
레벨 4:  G                ← 리프, 깊이 3, 높이 0
```

- 트리의 높이 = 루트의 높이 = 3
- 노드 A의 차수 = 2 (자식 B, C)
- 노드 C의 차수 = 1 (자식 F)

### 이진 트리(Binary Tree)

모든 노드의 자식이 **최대 2개**인 트리. 왼쪽 자식과 오른쪽 자식으로 구분한다.

**이진 트리의 종류:**

```
[완전 이진 트리]         [포화 이진 트리]        [편향 이진 트리]
      A                      A                    A
     / \                   /   \                    \
    B   C                 B     C                    B
   / \                   / \   / \                    \
  D   E                 D   E F   G                    C
                                                        \
                                                         D
```

- **완전 이진 트리**: 마지막 레벨을 제외하고 모든 레벨이 채워지고, 마지막 레벨은 왼쪽부터 채워진 트리
- **포화 이진 트리**: 모든 레벨이 완전히 채워진 트리
- **편향 이진 트리**: 한쪽으로만 치우친 트리 (사실상 연결 리스트)

### 이진 트리 순회

트리의 모든 노드를 한 번씩 방문하는 방법. 4가지 순회 방법이 있다.

```
        1
       / \
      2   3
     / \
    4   5
```

| 순회 방식 | 방문 순서 | 결과 | 기억법 |
|-----------|-----------|------|--------|
| 전위(Preorder) | **루트** → 왼쪽 → 오른쪽 | 1, 2, 4, 5, 3 | 루트가 **앞(Pre)** |
| 중위(Inorder) | 왼쪽 → **루트** → 오른쪽 | 4, 2, 5, 1, 3 | 루트가 **중간(In)** |
| 후위(Postorder) | 왼쪽 → 오른쪽 → **루트** | 4, 5, 2, 3, 1 | 루트가 **뒤(Post)** |
| 레벨(Level-order) | 레벨 순서대로 | 1, 2, 3, 4, 5 | 위에서 아래로 |

> **비유**: 전위 순회는 "사장이 먼저 인사하고 부하직원 순", 후위 순회는 "부하직원이 먼저 보고하고 사장이 마지막에 확인"

### 이진 탐색 트리(BST)

**왼쪽 서브트리의 모든 값 &lt; 루트 &lt; 오른쪽 서브트리의 모든 값**을 만족하는 이진 트리.

```
        8
       / \
      3   10
     / \    \
    1   6   14
       / \  /
      4  7 13
```

- 왼쪽 서브트리의 모든 값(1, 3, 4, 6, 7) &lt; 8 &lt; 오른쪽 서브트리의 모든 값(10, 13, 14)
- 이 성질은 모든 서브트리에 재귀적으로 적용된다
- **중위 순회 시 정렬된 순서**: 1, 3, 4, 6, 7, 8, 10, 13, 14

**BST 연산의 시간 복잡도:**

| 연산 | 평균 | 최악 (편향 트리) |
|------|------|-----------------|
| 검색 | O(log n) | O(n) |
| 삽입 | O(log n) | O(n) |
| 삭제 | O(log n) | O(n) |

**BST 삭제의 3가지 경우:**

1. **리프 노드 삭제**: 그냥 제거
2. **자식이 1개인 노드 삭제**: 자식을 삭제 노드 위치로 올림
3. **자식이 2개인 노드 삭제**: 오른쪽 서브트리의 최솟값(중위 후속자) 또는 왼쪽 서브트리의 최댓값(중위 선행자)으로 교체

### AVL 트리 — 균형 이진 탐색 트리

BST가 편향되면 O(n)으로 성능이 떨어진다. AVL 트리는 **모든 노드에서 왼쪽/오른쪽 서브트리의 높이 차이(균형 인수)가 -1, 0, 1**이 되도록 유지한다.

**균형 인수(Balance Factor)** = 왼쪽 서브트리 높이 - 오른쪽 서브트리 높이

불균형이 발생하면 **회전(Rotation)**으로 해결:

```
[LL 회전 - 단순 우회전]          [RR 회전 - 단순 좌회전]
    z                               z
   / \                             / \
  y   T4    →     y               T1   y      →      y
 / \             / \                  / \            / \
x   T3          x   z               T2   x         z   x
               / \ / \                  / \        / \ / \
              T1 T2 T3 T4             T3  T4      T1 T2 T3 T4

[LR 회전 - 왼쪽-오른쪽]          [RL 회전 - 오른쪽-왼쪽]
    z                               z
   / \                             / \
  x   T4   →      y              T1   x     →      y
 / \              / \                 / \          / \
T1   y           x   z              y   T4        z   x
    / \         / \ / \            / \           / \ / \
   T2  T3     T1 T2 T3 T4        T2  T3       T1 T2 T3 T4
```

## 코드 예시

### 이진 트리 노드 및 순회 구현

```python
class TreeNode:
    """이진 트리 노드"""
    def __init__(self, val):
        self.val = val
        self.left = None   # 왼쪽 자식
        self.right = None  # 오른쪽 자식


def preorder(node):
    """전위 순회: 루트 → 왼쪽 → 오른쪽"""
    if node is None:
        return []
    result = [node.val]                    # 루트 방문
    result += preorder(node.left)          # 왼쪽 서브트리
    result += preorder(node.right)         # 오른쪽 서브트리
    return result


def inorder(node):
    """중위 순회: 왼쪽 → 루트 → 오른쪽 (BST에서는 정렬 순서)"""
    if node is None:
        return []
    result = inorder(node.left)            # 왼쪽 서브트리
    result += [node.val]                    # 루트 방문
    result += inorder(node.right)          # 오른쪽 서브트리
    return result


def postorder(node):
    """후위 순회: 왼쪽 → 오른쪽 → 루트"""
    if node is None:
        return []
    result = postorder(node.left)          # 왼쪽 서브트리
    result += postorder(node.right)        # 오른쪽 서브트리
    result += [node.val]                    # 루트 방문
    return result


from collections import deque

def level_order(root):
    """레벨 순회: BFS 방식으로 레벨 단위 방문"""
    if root is None:
        return []
    result = []
    queue = deque([root])                  # 큐에 루트부터 시작
    while queue:
        node = queue.popleft()             # 큐에서 꺼냄
        result.append(node.val)            # 방문
        if node.left:
            queue.append(node.left)        # 왼쪽 자식 큐에 추가
        if node.right:
            queue.append(node.right)       # 오른쪽 자식 큐에 추가
    return result


# === 트리 생성 및 순회 테스트 ===
#         1
#        / \
#       2   3
#      / \
#     4   5
root = TreeNode(1)
root.left = TreeNode(2)
root.right = TreeNode(3)
root.left.left = TreeNode(4)
root.left.right = TreeNode(5)

print("전위 순회:", preorder(root))       # [1, 2, 4, 5, 3]
print("중위 순회:", inorder(root))        # [4, 2, 5, 1, 3]
print("후위 순회:", postorder(root))      # [4, 5, 2, 3, 1]
print("레벨 순회:", level_order(root))    # [1, 2, 3, 4, 5]
```

### BST 삽입, 검색, 삭제 구현

```python
class BST:
    """이진 탐색 트리"""
    def __init__(self):
        self.root = None

    def insert(self, val):
        """값을 BST에 삽입"""
        self.root = self._insert(self.root, val)

    def _insert(self, node, val):
        # 빈 자리를 찾으면 새 노드 생성
        if node is None:
            return TreeNode(val)
        # 값이 작으면 왼쪽으로
        if val < node.val:
            node.left = self._insert(node.left, val)
        # 값이 크면 오른쪽으로
        elif val > node.val:
            node.right = self._insert(node.right, val)
        # 중복 값은 무시 (또는 정책에 따라 처리)
        return node

    def search(self, val):
        """값이 BST에 존재하는지 검색"""
        return self._search(self.root, val)

    def _search(self, node, val):
        if node is None:
            return False                   # 못 찾음
        if val == node.val:
            return True                    # 찾음!
        elif val < node.val:
            return self._search(node.left, val)   # 왼쪽 탐색
        else:
            return self._search(node.right, val)  # 오른쪽 탐색

    def delete(self, val):
        """값을 BST에서 삭제"""
        self.root = self._delete(self.root, val)

    def _delete(self, node, val):
        if node is None:
            return None

        if val < node.val:
            node.left = self._delete(node.left, val)
        elif val > node.val:
            node.right = self._delete(node.right, val)
        else:
            # 삭제할 노드를 찾음
            # Case 1: 리프 노드 또는 자식이 1개
            if node.left is None:
                return node.right
            if node.right is None:
                return node.left
            # Case 2: 자식이 2개 → 중위 후속자(오른쪽 서브트리 최솟값)로 교체
            successor = self._find_min(node.right)
            node.val = successor.val
            node.right = self._delete(node.right, successor.val)

        return node

    def _find_min(self, node):
        """서브트리에서 최솟값 노드 찾기 (가장 왼쪽 노드)"""
        while node.left:
            node = node.left
        return node

    def inorder(self):
        """중위 순회로 정렬된 결과 반환"""
        return inorder(self.root)


# === BST 사용 예시 ===
bst = BST()
for val in [8, 3, 10, 1, 6, 14, 4, 7, 13]:
    bst.insert(val)

print("중위 순회:", bst.inorder())        # [1, 3, 4, 6, 7, 8, 10, 13, 14]
print("6 검색:", bst.search(6))           # True
print("99 검색:", bst.search(99))         # False

bst.delete(3)  # 자식 2개인 노드 삭제
print("3 삭제 후:", bst.inorder())        # [1, 4, 6, 7, 8, 10, 13, 14]
```

### 트리 높이 및 노드 수 계산

```python
def tree_height(node):
    """트리의 높이 계산"""
    if node is None:
        return -1  # 빈 트리의 높이는 -1 (노드가 없으므로)
    left_h = tree_height(node.left)
    right_h = tree_height(node.right)
    return 1 + max(left_h, right_h)


def count_nodes(node):
    """트리의 노드 수 계산"""
    if node is None:
        return 0
    return 1 + count_nodes(node.left) + count_nodes(node.right)


def is_bst(node, min_val=float('-inf'), max_val=float('inf')):
    """BST 유효성 검증"""
    if node is None:
        return True
    if not (min_val < node.val < max_val):
        return False
    return (is_bst(node.left, min_val, node.val) and
            is_bst(node.right, node.val, max_val))
```

## 정리

| 항목 | 내용 |
|------|------|
| 트리 | 계층적 비선형 자료구조, 루트에서 리프까지 단일 경로 |
| 이진 트리 | 각 노드의 자식이 최대 2개인 트리 |
| BST | 왼쪽 &lt; 루트 &lt; 오른쪽 성질을 만족하는 이진 트리 |
| 전위 순회 | 루트 → 왼쪽 → 오른쪽 |
| 중위 순회 | 왼쪽 → 루트 → 오른쪽 (BST에서 정렬 순서) |
| 후위 순회 | 왼쪽 → 오른쪽 → 루트 |
| 레벨 순회 | BFS 방식, 큐 사용 |
| BST 평균 시간 | 검색/삽입/삭제 모두 O(log n) |
| BST 최악 시간 | 편향 트리일 때 O(n) |
| AVL 트리 | 균형 인수 유지 (-1, 0, 1), 회전으로 균형 복구 |
