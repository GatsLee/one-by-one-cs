## 핵심 개념

### 해시 테이블이란?

해시 테이블은 **키(key)를 값(value)에 매핑**하는 자료구조다. 사전(dictionary)을 떠올려보자. "apple"이라는 단어(키)를 찾으면 "사과"라는 뜻(값)이 나온다. 중요한 것은, 사전에서 단어를 찾을 때 처음부터 한 페이지씩 넘기지 않고 **바로 해당 페이지를 펼친다**는 점이다.

해시 테이블도 마찬가지다. 키를 받으면 **해시 함수**를 통해 배열의 인덱스를 계산하고, 그 위치에 바로 접근한다.

```
키 "apple"
    │
    v
┌────────────┐
│ 해시 함수   │  → hash("apple") = 3
└────────────┘
    │
    v
배열 인덱스 3에 직접 접근!

인덱스:  [0]      [1]      [2]      [3]        [4]
       +--------+--------+--------+----------+--------+
       |        | "dog"  |        | "apple"  | "cat"  |
       |  None  | ="개"  |  None  | ="사과"  | ="고양이"|
       +--------+--------+--------+----------+--------+
```

### 해시 함수 (Hash Function)

해시 함수는 **임의의 크기의 데이터를 고정 크기의 값으로 변환**하는 함수다.

좋은 해시 함수의 조건:
1. **결정적(Deterministic)**: 같은 입력은 항상 같은 출력
2. **균일 분포(Uniform Distribution)**: 출력이 골고루 퍼져야 함
3. **빠른 계산**: 해시 값 계산이 빨라야 의미가 있음

```
간단한 해시 함수 예시 (문자열):

hash("cat") = (99 + 97 + 116) % 10 = 312 % 10 = 2
hash("dog") = (100 + 111 + 103) % 10 = 314 % 10 = 4
hash("ant") = (97 + 110 + 116) % 10 = 323 % 10 = 3

각 문자의 ASCII 값을 더한 후, 배열 크기(10)로 나머지 연산
```

**Python의 `hash()` 내장 함수:**
```python
print(hash("hello"))     # 큰 정수값 (실행마다 다를 수 있음)
print(hash(42))          # 42
print(hash((1, 2, 3)))   # 튜플은 해시 가능
# hash([1, 2, 3])        # TypeError! 리스트는 해시 불가 (가변 객체)
```

중요: **가변(mutable) 객체는 해시할 수 없다.** 리스트, 딕셔너리, 집합 등은 내용이 변할 수 있으므로 해시 키로 사용할 수 없다. 이것이 Python에서 딕셔너리의 키로 리스트를 쓸 수 없는 이유다.

### 해시 충돌 (Hash Collision)

서로 다른 키가 **같은 해시 값(인덱스)**을 가지는 경우를 충돌이라 한다. 비둘기집 원리에 의해 충돌은 피할 수 없다. (키의 종류가 배열 크기보다 많으면 반드시 충돌 발생)

```
hash("cat") = 2
hash("act") = 2  ← 충돌! 같은 인덱스!

이름은 다르지만 같은 아파트 호수를 배정받은 셈
```

충돌을 해결하는 두 가지 주요 방법이 있다.

### 충돌 해결 1: 체이닝 (Chaining)

각 배열 슬롯에 **연결 리스트**를 두어, 같은 인덱스에 여러 키-값 쌍을 저장한다.

```
체이닝 방식:

인덱스 [0]: None
인덱스 [1]: None
인덱스 [2]: ("cat", "고양이") → ("act", "행동") → None
인덱스 [3]: ("ant", "개미") → None
인덱스 [4]: ("dog", "개") → None

인덱스 2에서 "cat"을 검색할 때:
→ 연결 리스트를 순회하며 키가 "cat"인 노드를 찾음
```

**장점:** 구현이 간단하고, 적재율이 높아도 동작한다
**단점:** 추가 메모리 필요 (연결 리스트 포인터), 캐시 효율 낮음

### 충돌 해결 2: 개방 주소법 (Open Addressing)

충돌 시 **다른 빈 슬롯을 찾아** 저장한다. 배열 안에서만 해결하므로 추가 메모리가 필요 없다.

**선형 탐사 (Linear Probing):**
충돌 발생 시 다음 칸, 그 다음 칸... 순서대로 빈 자리를 찾는다.

```
hash("cat") = 2 → 인덱스 2에 저장

hash("act") = 2 → 인덱스 2 충돌!
                → 인덱스 3 확인 → 비어있음! → 인덱스 3에 저장

hash("bat") = 2 → 인덱스 2 충돌!
                → 인덱스 3 충돌!
                → 인덱스 4 확인 → 비어있음! → 인덱스 4에 저장

인덱스:  [0]     [1]     [2]       [3]       [4]
       +-------+-------+---------+---------+---------+
       | None  | None  | "cat"   | "act"   | "bat"   |
       +-------+-------+---------+---------+---------+
```

**문제점 — 클러스터링(Clustering):**
연속된 슬롯이 채워지면서 탐색 시간이 길어지는 현상. 데이터가 한 곳에 뭉치게 된다.

**이중 해싱 (Double Hashing):**
두 번째 해시 함수를 사용하여 탐사 간격을 결정한다. 클러스터링을 줄여준다.
```
다음 위치 = (현재 위치 + h2(key)) % 배열크기
```

### Python dict의 내부 구조

Python의 `dict`는 **개방 주소법**을 사용한다 (정확히는 랜덤 탐사의 변형).

핵심 특성:
- 해시 테이블의 **적재율(load factor)**이 2/3을 넘으면 테이블 크기를 확장
- Python 3.7부터 딕셔너리는 **삽입 순서를 보장** (내부적으로 별도의 순서 배열 유지)
- 키는 반드시 **해시 가능(hashable)**해야 함 (불변 객체: str, int, tuple 등)

```
Python dict 내부 (간소화):

해시 테이블 (인덱스 → 엔트리 배열의 위치):
[-, 1, -, 0, -, 2, -, -]

엔트리 배열 (삽입 순서대로):
[("name", "Alice"), ("age", 30), ("city", "Seoul")]

"name" 검색:
1. hash("name") % 8 = 3
2. 해시 테이블[3] = 0
3. 엔트리 배열[0] = ("name", "Alice") ✓
```

### 시간 복잡도

| 연산 | 평균 | 최악 |
|------|------|------|
| 검색 (조회) | O(1) | O(n) |
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |

**평균 O(1)인 이유:** 해시 함수가 키를 균일하게 분포시키면 충돌이 적어서, 대부분 한 번의 계산으로 원하는 위치에 접근한다.

**최악 O(n)인 경우:** 모든 키가 같은 해시 값을 가져 하나의 체인에 몰리면, 연결 리스트를 순회해야 하므로 O(n)이 된다. 하지만 좋은 해시 함수를 사용하면 이런 경우는 극히 드물다.

## 정리

| 항목 | 설명 |
|------|------|
| 해시 테이블 | 키를 해시 함수로 인덱스로 변환하여 O(1) 접근하는 자료구조 |
| 해시 함수 | 임의 크기의 데이터를 고정 크기 값으로 변환하는 함수 |
| 해시 충돌 | 서로 다른 키가 같은 해시 값을 가지는 현상 (필연적) |
| 체이닝 | 같은 인덱스에 연결 리스트로 여러 항목 저장 |
| 선형 탐사 | 충돌 시 다음 빈 칸을 순차적으로 탐색 |
| 적재율 | `원소 수 / 테이블 크기`. 높을수록 충돌 증가 |
| Python dict | 개방 주소법 사용, 적재율 2/3 초과 시 확장, 삽입 순서 보장 (3.7+) |
| Python set | dict와 같은 해시 테이블이지만 값 없이 키만 저장 |
| 평균 시간 복잡도 | 검색/삽입/삭제 모두 O(1) |
| 최악 시간 복잡도 | 모든 키가 충돌하면 O(n) |
| 해시 가능 조건 | 불변(immutable) 객체만 해시 가능 (str, int, tuple 등) |
