## 핵심 개념

### 해시 테이블이란?

해시 테이블은 **키(key)를 값(value)에 매핑**하는 자료구조다. 사전(dictionary)을 떠올려보자. "apple"이라는 단어(키)를 찾으면 "사과"라는 뜻(값)이 나온다. 중요한 것은, 사전에서 단어를 찾을 때 처음부터 한 페이지씩 넘기지 않고 **바로 해당 페이지를 펼친다**는 점이다.

해시 테이블도 마찬가지다. 키를 받으면 **해시 함수**를 통해 배열의 인덱스를 계산하고, 그 위치에 바로 접근한다.

```
키 "apple"
    │
    v
┌────────────┐
│ 해시 함수   │  → hash("apple") = 3
└────────────┘
    │
    v
배열 인덱스 3에 직접 접근!

인덱스:  [0]      [1]      [2]      [3]        [4]
       +--------+--------+--------+----------+--------+
       |        | "dog"  |        | "apple"  | "cat"  |
       |  None  | ="개"  |  None  | ="사과"  | ="고양이"|
       +--------+--------+--------+----------+--------+
```

### 해시 함수 (Hash Function)

해시 함수는 **임의의 크기의 데이터를 고정 크기의 값으로 변환**하는 함수다.

좋은 해시 함수의 조건:
1. **결정적(Deterministic)**: 같은 입력은 항상 같은 출력
2. **균일 분포(Uniform Distribution)**: 출력이 골고루 퍼져야 함
3. **빠른 계산**: 해시 값 계산이 빨라야 의미가 있음

```
간단한 해시 함수 예시 (문자열):

hash("cat") = (99 + 97 + 116) % 10 = 312 % 10 = 2
hash("dog") = (100 + 111 + 103) % 10 = 314 % 10 = 4
hash("ant") = (97 + 110 + 116) % 10 = 323 % 10 = 3

각 문자의 ASCII 값을 더한 후, 배열 크기(10)로 나머지 연산
```

**Python의 `hash()` 내장 함수:**
```python
print(hash("hello"))     # 큰 정수값 (실행마다 다를 수 있음)
print(hash(42))          # 42
print(hash((1, 2, 3)))   # 튜플은 해시 가능
# hash([1, 2, 3])        # TypeError! 리스트는 해시 불가 (가변 객체)
```

중요: **가변(mutable) 객체는 해시할 수 없다.** 리스트, 딕셔너리, 집합 등은 내용이 변할 수 있으므로 해시 키로 사용할 수 없다. 이것이 Python에서 딕셔너리의 키로 리스트를 쓸 수 없는 이유다.

### 해시 충돌 (Hash Collision)

서로 다른 키가 **같은 해시 값(인덱스)**을 가지는 경우를 충돌이라 한다. 비둘기집 원리에 의해 충돌은 피할 수 없다. (키의 종류가 배열 크기보다 많으면 반드시 충돌 발생)

```
hash("cat") = 2
hash("act") = 2  ← 충돌! 같은 인덱스!

이름은 다르지만 같은 아파트 호수를 배정받은 셈
```

충돌을 해결하는 두 가지 주요 방법이 있다.

### 충돌 해결 1: 체이닝 (Chaining)

각 배열 슬롯에 **연결 리스트**를 두어, 같은 인덱스에 여러 키-값 쌍을 저장한다.

```
체이닝 방식:

인덱스 [0]: None
인덱스 [1]: None
인덱스 [2]: ("cat", "고양이") → ("act", "행동") → None
인덱스 [3]: ("ant", "개미") → None
인덱스 [4]: ("dog", "개") → None

인덱스 2에서 "cat"을 검색할 때:
→ 연결 리스트를 순회하며 키가 "cat"인 노드를 찾음
```

**장점:** 구현이 간단하고, 적재율이 높아도 동작한다
**단점:** 추가 메모리 필요 (연결 리스트 포인터), 캐시 효율 낮음

### 충돌 해결 2: 개방 주소법 (Open Addressing)

충돌 시 **다른 빈 슬롯을 찾아** 저장한다. 배열 안에서만 해결하므로 추가 메모리가 필요 없다.

**선형 탐사 (Linear Probing):**
충돌 발생 시 다음 칸, 그 다음 칸... 순서대로 빈 자리를 찾는다.

```
hash("cat") = 2 → 인덱스 2에 저장

hash("act") = 2 → 인덱스 2 충돌!
                → 인덱스 3 확인 → 비어있음! → 인덱스 3에 저장

hash("bat") = 2 → 인덱스 2 충돌!
                → 인덱스 3 충돌!
                → 인덱스 4 확인 → 비어있음! → 인덱스 4에 저장

인덱스:  [0]     [1]     [2]       [3]       [4]
       +-------+-------+---------+---------+---------+
       | None  | None  | "cat"   | "act"   | "bat"   |
       +-------+-------+---------+---------+---------+
```

**문제점 — 클러스터링(Clustering):**
연속된 슬롯이 채워지면서 탐색 시간이 길어지는 현상. 데이터가 한 곳에 뭉치게 된다.

**이중 해싱 (Double Hashing):**
두 번째 해시 함수를 사용하여 탐사 간격을 결정한다. 클러스터링을 줄여준다.
```
다음 위치 = (현재 위치 + h2(key)) % 배열크기
```

### Python dict의 내부 구조

Python의 `dict`는 **개방 주소법**을 사용한다 (정확히는 랜덤 탐사의 변형).

핵심 특성:
- 해시 테이블의 **적재율(load factor)**이 2/3을 넘으면 테이블 크기를 확장
- Python 3.7부터 딕셔너리는 **삽입 순서를 보장** (내부적으로 별도의 순서 배열 유지)
- 키는 반드시 **해시 가능(hashable)**해야 함 (불변 객체: str, int, tuple 등)

```
Python dict 내부 (간소화):

해시 테이블 (인덱스 → 엔트리 배열의 위치):
[-, 1, -, 0, -, 2, -, -]

엔트리 배열 (삽입 순서대로):
[("name", "Alice"), ("age", 30), ("city", "Seoul")]

"name" 검색:
1. hash("name") % 8 = 3
2. 해시 테이블[3] = 0
3. 엔트리 배열[0] = ("name", "Alice") ✓
```

### 시간 복잡도

| 연산 | 평균 | 최악 |
|------|------|------|
| 검색 (조회) | O(1) | O(n) |
| 삽입 | O(1) | O(n) |
| 삭제 | O(1) | O(n) |

**평균 O(1)인 이유:** 해시 함수가 키를 균일하게 분포시키면 충돌이 적어서, 대부분 한 번의 계산으로 원하는 위치에 접근한다.

**최악 O(n)인 경우:** 모든 키가 같은 해시 값을 가져 하나의 체인에 몰리면, 연결 리스트를 순회해야 하므로 O(n)이 된다. 하지만 좋은 해시 함수를 사용하면 이런 경우는 극히 드물다.

## 코드 예시

### 1. 해시 테이블 직접 구현 (체이닝)

```python
class HashTable:
    """체이닝 방식의 해시 테이블"""

    def __init__(self, capacity=8):
        self._capacity = capacity
        # 각 슬롯은 리스트 (체인)로 초기화
        self._buckets = [[] for _ in range(capacity)]
        self._size = 0

    def _hash(self, key):
        """해시 함수: 키를 인덱스로 변환"""
        return hash(key) % self._capacity

    def put(self, key, value):
        """키-값 쌍 삽입/갱신 - 평균 O(1)"""
        index = self._hash(key)
        bucket = self._buckets[index]

        # 이미 존재하는 키인지 확인 (갱신)
        for i, (k, v) in enumerate(bucket):
            if k == key:
                bucket[i] = (key, value)  # 값 갱신
                return

        # 새 키 삽입
        bucket.append((key, value))
        self._size += 1

        # 적재율 체크 → 필요시 확장
        if self._size / self._capacity > 0.75:
            self._resize()

    def get(self, key, default=None):
        """키로 값 조회 - 평균 O(1)"""
        index = self._hash(key)
        bucket = self._buckets[index]

        for k, v in bucket:
            if k == key:
                return v
        return default  # 키가 없으면 기본값 반환

    def delete(self, key):
        """키-값 쌍 삭제 - 평균 O(1)"""
        index = self._hash(key)
        bucket = self._buckets[index]

        for i, (k, v) in enumerate(bucket):
            if k == key:
                del bucket[i]
                self._size -= 1
                return True
        raise KeyError(f"키 '{key}'를 찾을 수 없습니다")

    def _resize(self):
        """테이블 크기 2배 확장"""
        old_buckets = self._buckets
        self._capacity *= 2
        self._buckets = [[] for _ in range(self._capacity)]
        self._size = 0

        # 모든 기존 항목을 새 테이블에 재해싱
        for bucket in old_buckets:
            for key, value in bucket:
                self.put(key, value)

    def __contains__(self, key):
        """'in' 연산자 지원"""
        return self.get(key, object()) is not object()

    def __len__(self):
        return self._size

    def __repr__(self):
        items = []
        for bucket in self._buckets:
            for k, v in bucket:
                items.append(f"{k!r}: {v!r}")
        return "{" + ", ".join(items) + "}"


# === 사용 예시 ===
ht = HashTable()
ht.put("name", "Alice")
ht.put("age", 30)
ht.put("city", "Seoul")

print(ht.get("name"))     # Alice
print(ht.get("age"))      # 30
print(ht.get("없는키", "기본값"))  # 기본값

ht.put("age", 31)  # 값 갱신
print(ht.get("age"))      # 31

ht.delete("city")
print(ht)  # {'name': 'Alice', 'age': 31}
```

### 2. 해시 테이블 직접 구현 (선형 탐사)

```python
class LinearProbingHashTable:
    """선형 탐사 방식의 해시 테이블"""
    _DELETED = object()  # 삭제 표시용 센티넬 값

    def __init__(self, capacity=8):
        self._capacity = capacity
        self._keys = [None] * capacity
        self._values = [None] * capacity
        self._size = 0

    def _hash(self, key):
        return hash(key) % self._capacity

    def put(self, key, value):
        """삽입: 충돌 시 다음 빈 칸을 찾아감"""
        if self._size / self._capacity > 0.5:
            self._resize()

        index = self._hash(key)
        while True:
            if self._keys[index] is None or self._keys[index] is self._DELETED:
                # 빈 칸 또는 삭제된 칸에 삽입
                self._keys[index] = key
                self._values[index] = value
                self._size += 1
                return
            elif self._keys[index] == key:
                # 이미 존재하는 키 → 값 갱신
                self._values[index] = value
                return
            # 충돌! 다음 칸으로 이동
            index = (index + 1) % self._capacity

    def get(self, key, default=None):
        """조회: 해시 위치부터 순차 탐색"""
        index = self._hash(key)
        start = index  # 무한 루프 방지용

        while self._keys[index] is not None:
            if self._keys[index] == key:
                return self._values[index]
            index = (index + 1) % self._capacity
            if index == start:
                break  # 한 바퀴 돌았으면 없음
        return default

    def delete(self, key):
        """
        삭제: 삭제 표시(tombstone)를 남김

        왜 None으로 비우면 안 되는가?
        → 뒤에 같은 해시의 원소가 있을 때 탐색 체인이 끊김!

        예) hash(A)=hash(B)=2, A는 인덱스2, B는 인덱스3에 저장
            A를 None으로 삭제하면 B를 검색할 때 인덱스2에서 None을 만나
            "없음"으로 판단해버림 → B를 찾을 수 없게 됨!
        """
        index = self._hash(key)
        while self._keys[index] is not None:
            if self._keys[index] == key:
                self._keys[index] = self._DELETED  # tombstone
                self._values[index] = None
                self._size -= 1
                return
            index = (index + 1) % self._capacity
        raise KeyError(f"키 '{key}'를 찾을 수 없습니다")

    def _resize(self):
        """크기 2배 확장"""
        old_keys, old_values = self._keys, self._values
        self._capacity *= 2
        self._keys = [None] * self._capacity
        self._values = [None] * self._capacity
        self._size = 0
        for k, v in zip(old_keys, old_values):
            if k is not None and k is not self._DELETED:
                self.put(k, v)

    def __repr__(self):
        items = []
        for k, v in zip(self._keys, self._values):
            if k is not None and k is not self._DELETED:
                items.append(f"{k!r}: {v!r}")
        return "{" + ", ".join(items) + "}"


# === 사용 예시 ===
ht = LinearProbingHashTable()
ht.put("apple", 1)
ht.put("banana", 2)
ht.put("cherry", 3)
print(ht.get("banana"))  # 2
ht.delete("banana")
print(ht.get("banana"))  # None
print(ht.get("cherry"))  # 3 (삭제 표시 덕분에 탐색 체인 유지)
```

### 3. 실전 활용: 카운팅 (빈도수 세기)

```python
def count_frequency(items):
    """원소의 빈도수를 센다 - O(n)"""
    counter = {}
    for item in items:
        # 방법 1: dict.get() 사용
        counter[item] = counter.get(item, 0) + 1
    return counter

# 테스트
words = ["apple", "banana", "apple", "cherry", "banana", "apple"]
print(count_frequency(words))
# {'apple': 3, 'banana': 2, 'cherry': 1}

# 방법 2: collections.Counter 사용 (더 간편)
from collections import Counter
print(Counter(words))
# Counter({'apple': 3, 'banana': 2, 'cherry': 1})

# Counter의 유용한 메서드
c = Counter(words)
print(c.most_common(2))  # [('apple', 3), ('banana', 2)] — 가장 많은 2개
```

### 4. 실전 활용: 중복 검사

```python
def has_duplicate(nums: list) -> bool:
    """리스트에 중복 원소가 있는지 확인 - O(n)"""
    seen = set()  # set도 내부적으로 해시 테이블
    for num in nums:
        if num in seen:   # O(1) 검색
            return True
        seen.add(num)     # O(1) 삽입
    return False

print(has_duplicate([1, 2, 3, 4, 5]))     # False
print(has_duplicate([1, 2, 3, 2, 5]))     # True

# 비교: 브루트포스 O(n²)
def has_duplicate_brute(nums):
    """이중 루프로 중복 검사 - O(n²) - 비효율적!"""
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] == nums[j]:
                return True
    return False
```

### 5. 실전 활용: 두 수의 합 (Two Sum)

```python
def two_sum(nums: list, target: int) -> tuple:
    """
    두 수의 합이 target이 되는 인덱스 쌍을 찾는다 - O(n)

    해시 맵에 "이미 본 숫자 → 인덱스"를 저장하고,
    현재 숫자의 보수(target - current)가 이미 있는지 확인한다.
    """
    seen = {}  # 값 → 인덱스

    for i, num in enumerate(nums):
        complement = target - num  # 필요한 보수
        if complement in seen:     # O(1) 검색
            return (seen[complement], i)
        seen[num] = i              # 현재 숫자 기록

    return None  # 답이 없는 경우


# 테스트
nums = [2, 7, 11, 15]
print(two_sum(nums, 9))   # (0, 1) → nums[0] + nums[1] = 2 + 7 = 9

nums = [3, 2, 4]
print(two_sum(nums, 6))   # (1, 2) → nums[1] + nums[2] = 2 + 4 = 6
```

### 6. defaultdict와 setdefault

```python
from collections import defaultdict

# 그룹핑: 이름의 첫 글자별로 분류
names = ["Alice", "Bob", "Anna", "Charlie", "Brian", "Amy"]

# 방법 1: 일반 dict
groups = {}
for name in names:
    first = name[0]
    if first not in groups:
        groups[first] = []
    groups[first].append(name)
print(groups)  # {'A': ['Alice', 'Anna', 'Amy'], 'B': ['Bob', 'Brian'], 'C': ['Charlie']}

# 방법 2: setdefault
groups2 = {}
for name in names:
    groups2.setdefault(name[0], []).append(name)

# 방법 3: defaultdict (가장 깔끔)
groups3 = defaultdict(list)  # 없는 키 접근 시 빈 리스트 자동 생성
for name in names:
    groups3[name[0]].append(name)
print(dict(groups3))
```

## 정리

| 항목 | 설명 |
|------|------|
| 해시 테이블 | 키를 해시 함수로 인덱스로 변환하여 O(1) 접근하는 자료구조 |
| 해시 함수 | 임의 크기의 데이터를 고정 크기 값으로 변환하는 함수 |
| 해시 충돌 | 서로 다른 키가 같은 해시 값을 가지는 현상 (필연적) |
| 체이닝 | 같은 인덱스에 연결 리스트로 여러 항목 저장 |
| 선형 탐사 | 충돌 시 다음 빈 칸을 순차적으로 탐색 |
| 적재율 | `원소 수 / 테이블 크기`. 높을수록 충돌 증가 |
| Python dict | 개방 주소법 사용, 적재율 2/3 초과 시 확장, 삽입 순서 보장 (3.7+) |
| Python set | dict와 같은 해시 테이블이지만 값 없이 키만 저장 |
| 평균 시간 복잡도 | 검색/삽입/삭제 모두 O(1) |
| 최악 시간 복잡도 | 모든 키가 충돌하면 O(n) |
| 해시 가능 조건 | 불변(immutable) 객체만 해시 가능 (str, int, tuple 등) |
