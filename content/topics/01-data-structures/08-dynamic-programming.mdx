## 핵심 개념

### 동적 프로그래밍(DP)이란?

동적 프로그래밍(Dynamic Programming)은 **큰 문제를 작은 하위 문제로 나누어 풀되, 이미 계산한 결과를 저장해서 중복 계산을 피하는** 알고리즘 설계 기법이다.

> **비유**: 계단을 오르는 방법을 셀 때, 3층까지 올라가는 방법을 이미 알고 있다면, 4층은 3층에서 한 칸 오르는 것만 추가하면 된다. 매번 1층부터 다시 세지 않는다.

### DP의 두 가지 조건

DP를 적용하려면 문제가 다음 두 조건을 만족해야 한다.

```
      ┌──────────────────────────────────────────┐
      │         DP 적용 가능 조건                  │
      ├──────────────────────────────────────────┤
      │                                          │
      │  1. 최적 부분 구조 (Optimal Substructure) │
      │     → 최적 해가 부분 문제의 최적 해로     │
      │       구성될 수 있다                      │
      │                                          │
      │  2. 중복 부분 문제 (Overlapping            │
      │     Subproblems)                         │
      │     → 같은 부분 문제가 여러 번 반복된다   │
      │                                          │
      └──────────────────────────────────────────┘
```

**최적 부분 구조**: 전체 문제의 최적 해가 부분 문제의 최적 해를 포함한다.
- 예: 서울→부산 최단 경로가 서울→대전→부산이라면, 서울→대전도 최단 경로여야 한다.

**중복 부분 문제**: 재귀적으로 풀 때 같은 하위 문제가 반복해서 등장한다.
- 예: fib(5) 계산 시 fib(3)이 여러 번 계산된다.

### Top-down vs Bottom-up

| 방식 | 이름 | 설명 | 구현 |
|------|------|------|------|
| **Top-down** | 메모이제이션 | 재귀 + 결과 캐싱 | 재귀 함수 + dict/배열 |
| **Bottom-up** | 타뷸레이션 | 작은 문제부터 순차적으로 해결 | 반복문 + 테이블 |

```
Top-down (위 → 아래):          Bottom-up (아래 → 위):

fib(5)를 구하려면               fib(0)부터 차례로 구한다
  → fib(4)와 fib(3) 필요         fib(0) = 0
    → fib(3)과 fib(2) 필요       fib(1) = 1
      → ... (필요할 때 계산)      fib(2) = 1
                                  fib(3) = 2
이미 계산한 건 캐시에서 반환     fib(4) = 3
                                  fib(5) = 5
```

## 피보나치 수열: DP 입문

### 1단계: 순수 재귀 (비효율)

```python
def fib_naive(n):
    """순수 재귀 - O(2^n) 지수 시간"""
    if n <= 1:
        return n
    return fib_naive(n - 1) + fib_naive(n - 2)
```

```
fib(5)의 호출 트리 (중복 계산이 매우 많다):

                    fib(5)
                   /      \
              fib(4)       fib(3)
             /     \       /    \
         fib(3)  fib(2)  fib(2)  fib(1)
         /   \    / \     / \
     fib(2) fib(1) fib(1) fib(0) fib(1) fib(0)
      / \
  fib(1) fib(0)

fib(2)가 3번, fib(3)이 2번 중복 호출!
```

### 2단계: 메모이제이션 (Top-down)

```python
def fib_memo(n, memo=None):
    """메모이제이션 - O(n) 시간, O(n) 공간"""
    if memo is None:
        memo = {}
    if n <= 1:
        return n
    if n in memo:
        return memo[n]  # 이미 계산한 결과 반환
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]


# Python 데코레이터를 이용한 방법
from functools import lru_cache

@lru_cache(maxsize=None)
def fib_cached(n):
    """lru_cache 데코레이터 활용"""
    if n <= 1:
        return n
    return fib_cached(n - 1) + fib_cached(n - 2)
```

### 3단계: 타뷸레이션 (Bottom-up)

```python
def fib_tab(n):
    """타뷸레이션 - O(n) 시간, O(n) 공간"""
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[0] = 0
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]


def fib_optimized(n):
    """공간 최적화 - O(n) 시간, O(1) 공간"""
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        curr = prev1 + prev2
        prev2 = prev1
        prev1 = curr
    return prev1
```

### 성능 비교

| 방식 | 시간복잡도 | 공간복잡도 | 비고 |
|------|-----------|-----------|------|
| 순수 재귀 | O(2^n) | O(n) 스택 | 실용적이지 않음 |
| 메모이제이션 | O(n) | O(n) | 재귀 스택 + 캐시 |
| 타뷸레이션 | O(n) | O(n) | 반복문, 스택 오버플로 없음 |
| 공간 최적화 | O(n) | O(1) | 직전 두 값만 유지 |

## 0/1 배낭 문제 (0/1 Knapsack)

### 문제 정의

n개의 물건이 있고 각 물건은 무게(w)와 가치(v)를 가진다. 배낭의 용량 W가 주어질 때, **배낭에 넣을 수 있는 물건의 최대 가치**를 구하라. 각 물건은 넣거나(1) 안 넣거나(0)만 가능하다.

```
물건:  A(w=2, v=3)  B(w=3, v=4)  C(w=4, v=5)  D(w=5, v=6)
배낭 용량: W = 7

최적 해: A + C = 무게 6, 가치 8
또는     A + B = 무게 5, 가치 7
→ A + C가 최적 (가치 8)
```

### 상태 정의와 점화식

**상태**: `dp[i][w]` = 처음 i개의 물건으로 용량 w의 배낭을 채울 때의 최대 가치

**점화식**:

```
dp[i][w] = max(
    dp[i-1][w],              # i번째 물건을 안 넣는 경우
    dp[i-1][w-wi] + vi       # i번째 물건을 넣는 경우 (wi <= w일 때)
)
```

### 테이블 채우기 과정

```
물건: A(w=2,v=3), B(w=3,v=4), C(w=4,v=5)
배낭 용량 W=7

       w=0  w=1  w=2  w=3  w=4  w=5  w=6  w=7
i=0     0    0    0    0    0    0    0    0   (물건 없음)
i=A     0    0    3    3    3    3    3    3   (A만)
i=B     0    0    3    4    4    7    7    7   (A,B)
i=C     0    0    3    4    5    7    8    9   (A,B,C)

dp[3][7] = 9  →  B(w=3,v=4) + C(w=4,v=5)
```

### Python 구현

```python
def knapsack_01(weights, values, W):
    """
    0/1 배낭 문제 - O(nW) 시간, O(nW) 공간
    weights: 각 물건의 무게 리스트
    values: 각 물건의 가치 리스트
    W: 배낭 용량
    """
    n = len(weights)
    # dp 테이블 초기화
    dp = [[0] * (W + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        wi = weights[i - 1]  # i번째 물건의 무게
        vi = values[i - 1]   # i번째 물건의 가치
        for w in range(W + 1):
            dp[i][w] = dp[i - 1][w]  # 안 넣는 경우
            if wi <= w:
                # 넣는 경우와 비교
                dp[i][w] = max(dp[i][w], dp[i - 1][w - wi] + vi)

    return dp[n][W]


# 테스트
weights = [2, 3, 4, 5]
values = [3, 4, 5, 6]
W = 7
print(knapsack_01(weights, values, W))  # 8


def knapsack_01_optimized(weights, values, W):
    """공간 최적화 버전 - O(W) 공간"""
    n = len(weights)
    dp = [0] * (W + 1)

    for i in range(n):
        # 역순으로 순회해야 같은 물건을 두 번 사용하지 않음
        for w in range(W, weights[i] - 1, -1):
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])

    return dp[W]
```

## 최장 공통 부분 수열 (LCS)

### 문제 정의

두 문자열이 주어졌을 때, 두 문자열 모두에 포함되는 **가장 긴 부분 수열(subsequence)**을 찾는다. 부분 수열은 연속하지 않아도 되지만, 원래 순서는 유지해야 한다.

```
X = "ABCBDAB"
Y = "BDCAB"

공통 부분 수열: "B", "AB", "BCB", "BCAB", ...
최장 공통 부분 수열(LCS): "BCAB" (길이 4)
```

### 점화식

**상태**: `dp[i][j]` = X[0..i-1]과 Y[0..j-1]의 LCS 길이

```
              ┌ dp[i-1][j-1] + 1      if X[i-1] == Y[j-1]
dp[i][j] =   │
              └ max(dp[i-1][j], dp[i][j-1])  otherwise
```

### 테이블 채우기

```
X = "ABCB", Y = "BDCB"

       ""   B   D   C   B
  ""    0   0   0   0   0
  A     0   0   0   0   0
  B     0   1   1   1   1
  C     0   1   1   2   2
  B     0   1   1   2   3

LCS 길이 = dp[4][4] = 3
LCS = "BCB"
```

### Python 구현

```python
def lcs(X, Y):
    """
    최장 공통 부분 수열 - O(mn) 시간, O(mn) 공간
    m, n: 각 문자열의 길이
    """
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 테이블 채우기
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i - 1] == Y[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    # 역추적으로 LCS 문자열 복원
    result = []
    i, j = m, n
    while i > 0 and j > 0:
        if X[i - 1] == Y[j - 1]:
            result.append(X[i - 1])
            i -= 1
            j -= 1
        elif dp[i - 1][j] > dp[i][j - 1]:
            i -= 1
        else:
            j -= 1

    return dp[m][n], "".join(reversed(result))


# 테스트
length, sequence = lcs("ABCBDAB", "BDCAB")
print(f"LCS 길이: {length}")    # 4
print(f"LCS 문자열: {sequence}")  # BCAB
```

## 편집 거리 (Edit Distance)

### 문제 정의

두 문자열이 주어졌을 때, 하나를 다른 하나로 변환하는 데 필요한 **최소 편집 연산 횟수**를 구한다. 허용되는 연산은 삽입(Insert), 삭제(Delete), 교체(Replace)이다.

```
"kitten" → "sitting" 변환:

1. kitten → sitten   (k → s 교체)
2. sitten → sittin   (e → i 교체)
3. sittin → sitting  (g 삽입)

편집 거리 = 3
```

### 점화식

**상태**: `dp[i][j]` = X[0..i-1]을 Y[0..j-1]로 변환하는 최소 연산 수

```
              ┌ dp[i-1][j-1]              if X[i-1] == Y[j-1] (변환 불필요)
dp[i][j] =   │
              │     ┌ dp[i-1][j-1] + 1    교체 (Replace)
              └ min │ dp[i-1][j] + 1      삭제 (Delete)
                    └ dp[i][j-1] + 1      삽입 (Insert)
```

### 테이블 채우기

```
X = "CAT", Y = "CUT"

       ""   C   U   T
  ""    0   1   2   3
  C     1   0   1   2
  A     2   1   1   2
  T     3   2   2   1

편집 거리 = dp[3][3] = 1  (A → U 교체)
```

### Python 구현

```python
def edit_distance(X, Y):
    """
    편집 거리 (레벤슈타인 거리) - O(mn) 시간, O(mn) 공간
    """
    m, n = len(X), len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    # 기저 사례: 빈 문자열과의 거리
    for i in range(m + 1):
        dp[i][0] = i  # X[:i]를 빈 문자열로: i번 삭제
    for j in range(n + 1):
        dp[0][j] = j  # 빈 문자열을 Y[:j]로: j번 삽입

    # 테이블 채우기
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i - 1] == Y[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]  # 같으면 비용 0
            else:
                dp[i][j] = 1 + min(
                    dp[i - 1][j - 1],  # 교체
                    dp[i - 1][j],      # 삭제
                    dp[i][j - 1]       # 삽입
                )

    return dp[m][n]


# 테스트
print(edit_distance("kitten", "sitting"))  # 3
print(edit_distance("sunday", "saturday"))  # 3
```

### 활용 사례

| 분야 | 활용 |
|------|------|
| 자연어 처리 | 맞춤법 교정, 유사 단어 추천 |
| 생물정보학 | DNA 서열 비교 |
| 검색 엔진 | "Did you mean...?" 기능 |
| 버전 관리 | diff 알고리즘의 기초 |

## 최장 증가 부분 수열 (LIS)

### 문제 정의

수열에서 **순서를 유지하면서 값이 증가하는 가장 긴 부분 수열**을 찾는다.

```
수열: [10, 9, 2, 5, 3, 7, 101, 18]

증가 부분 수열 예:
  [2, 5, 7, 101]  ← 길이 4 (최장)
  [2, 5, 7, 18]   ← 길이 4 (최장)
  [2, 3, 7, 18]   ← 길이 4 (최장)

LIS 길이 = 4
```

### O(n^2) 기본 풀이

```python
def lis_basic(nums):
    """
    최장 증가 부분 수열 - O(n^2) 시간
    dp[i] = nums[i]로 끝나는 LIS의 길이
    """
    if not nums:
        return 0

    n = len(nums)
    dp = [1] * n  # 최소 자기 자신 1개

    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)


# 테스트
nums = [10, 9, 2, 5, 3, 7, 101, 18]
print(lis_basic(nums))  # 4
```

```
dp 테이블 구성 과정:

인덱스:  0    1    2    3    4    5    6     7
값:     10    9    2    5    3    7   101   18
dp:      1    1    1    2    2    3    4     4

dp[3]=2: nums[2]=2 < nums[3]=5 → dp[2]+1=2
dp[5]=3: nums[4]=3 < nums[5]=7 → dp[4]+1=3
dp[6]=4: nums[5]=7 < nums[6]=101 → dp[5]+1=4
```

### O(n log n) 이진탐색 최적화

핵심 아이디어: **tails 배열을 유지**한다. `tails[i]`는 길이 (i+1)인 증가 수열의 마지막 원소 중 가장 작은 값이다.

```python
import bisect

def lis_optimized(nums):
    """
    최장 증가 부분 수열 - O(n log n) 시간
    이진탐색으로 tails 배열 관리
    """
    if not nums:
        return 0

    tails = []  # tails[i] = 길이 (i+1) LIS의 최소 마지막 값

    for num in nums:
        # tails에서 num 이상인 첫 위치를 이진탐색
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)  # 새로운 최장 길이 발견
        else:
            tails[pos] = num   # 기존 값을 더 작은 값으로 교체

    return len(tails)


# 테스트
nums = [10, 9, 2, 5, 3, 7, 101, 18]
print(lis_optimized(nums))  # 4
```

```
tails 배열 변화 과정:

num=10:  tails=[10]          (추가)
num=9:   tails=[9]           (10을 9로 교체)
num=2:   tails=[2]           (9를 2로 교체)
num=5:   tails=[2, 5]        (추가)
num=3:   tails=[2, 3]        (5를 3으로 교체)
num=7:   tails=[2, 3, 7]     (추가)
num=101: tails=[2, 3, 7, 101] (추가)
num=18:  tails=[2, 3, 7, 18]  (101을 18로 교체)

최종 길이 = 4

주의: tails 자체가 LIS는 아니다! 길이만 정확하다.
```

### LIS 비교

| 방법 | 시간복잡도 | 공간복잡도 | LIS 복원 |
|------|-----------|-----------|---------|
| O(n^2) DP | O(n^2) | O(n) | 가능 (역추적) |
| O(n log n) 이진탐색 | O(n log n) | O(n) | 별도 처리 필요 |

## DP vs 그리디 비교

| 항목 | 동적 프로그래밍 (DP) | 그리디 알고리즘 |
|------|---------------------|----------------|
| 접근 방식 | 모든 부분 문제를 해결 | 현재 최선만 선택 |
| 최적성 보장 | 항상 최적 해 보장 | 문제에 따라 다름 |
| 시간복잡도 | 보통 더 느림 | 보통 더 빠름 |
| 필요 조건 | 최적 부분 구조 + 중복 부분 문제 | 탐욕 선택 속성 + 최적 부분 구조 |
| 예시 | 0/1 배낭, LCS, 편집거리 | 분할 배낭, 활동 선택, 허프만 |

> **핵심 차이**: DP는 "모든 경우를 체계적으로 탐색"하고, 그리디는 "매 순간 최선을 선택"한다. 그리디가 적용 가능한 문제는 DP보다 효율적이지만, 모든 문제에 그리디를 적용할 수 있는 것은 아니다.


---

## DP 문제 풀이 프레임워크

DP 문제를 체계적으로 풀기 위한 5단계:

```
1단계: 상태 정의
  → dp[i]가 무엇을 의미하는지 명확히 정의

2단계: 점화식 도출
  → dp[i]를 이전 상태(dp[i-1], dp[i-2], ...)로 표현

3단계: 기저 사례 설정
  → dp[0], dp[1] 등 초기값 결정

4단계: 채우기 순서 결정
  → Top-down (재귀) 또는 Bottom-up (반복)

5단계: 최종 답 위치 확인
  → dp[n]인지, max(dp)인지, dp[m][n]인지
```

## 정리

| 항목 | 내용 |
|------|------|
| DP 조건 | 최적 부분 구조 + 중복 부분 문제 |
| Top-down | 재귀 + 메모이제이션 (필요한 것만 계산) |
| Bottom-up | 반복문 + 테이블 (작은 것부터 전부 계산) |
| 0/1 배낭 | dp[i][w] = max(안 넣음, 넣음), O(nW) |
| LCS | 두 문자 같으면 대각선+1, 다르면 max(위, 왼쪽), O(mn) |
| 편집 거리 | 삽입/삭제/교체 중 최소, O(mn) |
| LIS | O(n^2) 기본, O(n log n) 이진탐색 최적화 |
| 공간 최적화 | 이전 행만 필요하면 1D 배열로 축소 가능 |
| DP vs 그리디 | DP는 모든 경우 탐색, 그리디는 매 순간 최선 |
