## 핵심 개념

### 정렬(Sorting)이란?

정렬은 데이터를 특정 기준(오름차순, 내림차순 등)에 따라 순서대로 배치하는 연산이다. 거의 모든 소프트웨어에서 핵심적으로 사용되며, 검색 효율 향상, 데이터 분석, 중복 제거 등 다양한 목적으로 활용된다.

> **비유**: 도서관에서 책을 제목의 가나다순으로 꽂아두면, 원하는 책을 빠르게 찾을 수 있다. 정렬은 이렇게 데이터를 "정돈"하는 과정이다.

### 정렬 알고리즘의 분류

```
                   정렬 알고리즘
                   /            \
          비교 기반              비비교 기반
         (Comparison)          (Non-comparison)
        /     |     \            /         \
    O(n^2)  O(n log n)       O(n+k)      O(d*(n+k))
   버블     머지소트        카운팅 정렬    기수 정렬
   선택     퀵소트
   삽입     힙소트
```

| 분류 | 설명 | 하한 |
|------|------|------|
| **비교 기반 정렬** | 원소 간 비교(&lt;, &gt;, =)로 순서 결정 | O(n log n) 이론적 하한 |
| **비비교 기반 정렬** | 원소의 값 자체를 이용 (자릿수, 빈도) | O(n) 가능 (조건부) |

비교 기반 정렬은 어떤 알고리즘을 사용하든 **최악의 경우 O(n log n)보다 빠를 수 없다**는 것이 결정 트리(Decision Tree) 모델로 증명되어 있다.

## O(n^2) 정렬 알고리즘

### 버블 정렬 (Bubble Sort)

인접한 두 원소를 비교하여 교환하는 방식. 매 패스마다 가장 큰 원소가 맨 뒤로 "떠오른다(bubble up)".

```
패스 1: [5, 3, 8, 1, 2]
         ^--^  → [3, 5, 8, 1, 2]
            ^--^  → [3, 5, 8, 1, 2] (변화 없음)
               ^--^  → [3, 5, 1, 8, 2]
                  ^--^  → [3, 5, 1, 2, 8]  ← 8이 제자리
```

```python
def bubble_sort(arr):
    """버블 정렬 - O(n^2)"""
    n = len(arr)
    for i in range(n):
        swapped = False  # 최적화: 교환이 없으면 이미 정렬됨
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break  # 이미 정렬 완료
    return arr
```

### 선택 정렬 (Selection Sort)

매 단계에서 **최솟값을 찾아** 현재 위치와 교환한다.

```
[5, 3, 8, 1, 2]
 최소=1 → [1, 3, 8, 5, 2]
    최소=2 → [1, 2, 8, 5, 3]
       최소=3 → [1, 2, 3, 5, 8]
          최소=5 → [1, 2, 3, 5, 8] (완료)
```

```python
def selection_sort(arr):
    """선택 정렬 - O(n^2), 불안정"""
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]  # 최솟값과 교환
    return arr
```

### 삽입 정렬 (Insertion Sort)

카드 정리하듯, 현재 원소를 **이미 정렬된 부분의 올바른 위치에 삽입**한다.

```
[5, |3, 8, 1, 2]   3을 삽입 → [3, 5, |8, 1, 2]
[3, 5, |8, 1, 2]   8을 삽입 → [3, 5, 8, |1, 2]
[3, 5, 8, |1, 2]   1을 삽입 → [1, 3, 5, 8, |2]
[1, 3, 5, 8, |2]   2를 삽입 → [1, 2, 3, 5, 8]
```

```python
def insertion_sort(arr):
    """삽입 정렬 - O(n^2), 안정, 거의 정렬된 데이터에 효율적"""
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]  # 오른쪽으로 이동
            j -= 1
        arr[j + 1] = key  # 올바른 위치에 삽입
    return arr
```

| 알고리즘 | 최선 | 평균 | 최악 | 안정성 | 특징 |
|----------|------|------|------|--------|------|
| 버블 정렬 | O(n) | O(n^2) | O(n^2) | 안정 | 구현 가장 단순 |
| 선택 정렬 | O(n^2) | O(n^2) | O(n^2) | **불안정** | 교환 횟수 최소 |
| 삽입 정렬 | O(n) | O(n^2) | O(n^2) | 안정 | 거의 정렬된 데이터에 빠름 |

## 머지소트 (Merge Sort)

### 분할 정복 전략

머지소트는 **분할 정복(Divide and Conquer)** 패러다임의 대표적인 예다.

1. **분할(Divide)**: 배열을 반으로 나눈다
2. **정복(Conquer)**: 각 반을 재귀적으로 정렬한다
3. **병합(Merge)**: 정렬된 두 반을 합친다

```
              [38, 27, 43, 3, 9, 82, 10]
             /                           \
      [38, 27, 43, 3]            [9, 82, 10]
       /           \              /         \
   [38, 27]    [43, 3]       [9, 82]      [10]
    /    \      /    \        /    \         |
  [38]  [27]  [43]  [3]    [9]  [82]      [10]
    \    /      \    /        \    /         |
   [27, 38]    [3, 43]       [9, 82]      [10]
       \           /              \         /
      [3, 27, 38, 43]          [9, 10, 82]
             \                     /
        [3, 9, 10, 27, 38, 43, 82]
```

### Python 구현

```python
def merge_sort(arr):
    """
    머지소트 - O(n log n), 안정 정렬
    추가 공간: O(n)
    """
    if len(arr) <= 1:
        return arr

    # 1단계: 분할
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])

    # 2단계: 병합
    return merge(left, right)


def merge(left, right):
    """정렬된 두 배열을 하나로 병합"""
    result = []
    i = j = 0

    # 두 배열에서 작은 값을 차례로 선택
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:  # <=로 안정성 유지
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # 남은 원소 추가
    result.extend(left[i:])
    result.extend(right[j:])
    return result


# 테스트
arr = [38, 27, 43, 3, 9, 82, 10]
print(merge_sort(arr))  # [3, 9, 10, 27, 38, 43, 82]
```

### 복잡도 분석

```
레벨 0:  n개 원소를 1번 병합       → O(n) 작업
레벨 1:  n/2개 원소를 2번 병합     → O(n) 작업
레벨 2:  n/4개 원소를 4번 병합     → O(n) 작업
  ...
레벨 k:  1개 원소를 n번            → O(n) 작업

총 레벨 수 = log₂(n)
총 시간복잡도 = O(n) × O(log n) = O(n log n)
```

| 항목 | 값 |
|------|-----|
| 시간복잡도 | 최선/평균/최악 모두 O(n log n) |
| 공간복잡도 | O(n) (병합 시 임시 배열 필요) |
| 안정 정렬 | 예 (같은 값의 원래 순서 유지) |
| 제자리 정렬 | 아니오 (추가 메모리 필요) |

## 퀵소트 (Quick Sort)

### 피벗과 파티션

퀵소트도 분할 정복 방식이지만, 머지소트와 달리 **파티션(분할)** 단계에서 대부분의 작업이 이루어진다.

1. **피벗(Pivot)** 선택: 배열에서 하나의 원소를 기준으로 선택
2. **파티션**: 피벗보다 작은 원소는 왼쪽, 큰 원소는 오른쪽으로 분류
3. **재귀**: 왼쪽 부분과 오른쪽 부분을 각각 재귀 정렬

```
피벗 = 5
[3, 7, 2, 5, 1, 8, 4]
       파티션
[3, 2, 1, 4] [5] [7, 8]
  왼쪽(< 5)   피벗  오른쪽(> 5)
      ↓                ↓
   재귀 정렬        재귀 정렬
[1, 2, 3, 4] [5] [7, 8]
→ [1, 2, 3, 4, 5, 7, 8]
```

### Lomuto 파티션 방식

```python
def lomuto_partition(arr, low, high):
    """
    Lomuto 파티션: 마지막 원소를 피벗으로 선택
    i는 피벗보다 작은 원소의 마지막 인덱스를 추적
    """
    pivot = arr[high]
    i = low - 1  # 작은 원소 영역의 경계

    for j in range(low, high):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]

    arr[i + 1], arr[high] = arr[high], arr[i + 1]
    return i + 1  # 피벗의 최종 위치


def quick_sort_lomuto(arr, low=0, high=None):
    """퀵소트 - Lomuto 파티션 방식"""
    if high is None:
        high = len(arr) - 1
    if low < high:
        pi = lomuto_partition(arr, low, high)
        quick_sort_lomuto(arr, low, pi - 1)
        quick_sort_lomuto(arr, pi + 1, high)
    return arr
```

### Hoare 파티션 방식

Hoare 방식은 양쪽 끝에서 포인터가 안쪽으로 이동하며, Lomuto보다 교환 횟수가 적다.

```python
def hoare_partition(arr, low, high):
    """
    Hoare 파티션: 첫 번째 원소를 피벗으로 선택
    두 포인터가 양끝에서 중앙으로 이동
    """
    pivot = arr[low]
    i = low - 1
    j = high + 1

    while True:
        i += 1
        while arr[i] < pivot:
            i += 1
        j -= 1
        while arr[j] > pivot:
            j -= 1
        if i >= j:
            return j
        arr[i], arr[j] = arr[j], arr[i]


def quick_sort_hoare(arr, low=0, high=None):
    """퀵소트 - Hoare 파티션 방식"""
    if high is None:
        high = len(arr) - 1
    if low < high:
        pi = hoare_partition(arr, low, high)
        quick_sort_hoare(arr, low, pi)
        quick_sort_hoare(arr, pi + 1, high)
    return arr
```

### Lomuto vs Hoare 비교

| 항목 | Lomuto | Hoare |
|------|--------|-------|
| 피벗 위치 | 마지막 원소 | 첫 번째 원소 |
| 평균 교환 횟수 | 더 많음 | 더 적음 (약 3배) |
| 구현 난이도 | 더 간단 | 약간 복잡 |
| 이미 정렬된 입력 | O(n^2) | O(n^2) |

### 최악의 경우와 최적화

퀵소트의 최악 O(n^2)는 매번 피벗이 **최솟값 또는 최댓값**일 때 발생한다 (이미 정렬된 배열 + 첫/마지막 원소 피벗).

```
최악의 경우: 이미 정렬된 [1, 2, 3, 4, 5], 피벗=마지막

[1, 2, 3, 4, | 5]   → 파티션: [] [5] [1,2,3,4]  ← 한쪽으로 치우침
[1, 2, 3, | 4]      → 파티션: [] [4] [1,2,3]
[1, 2, | 3]          → 파티션: [] [3] [1,2]
...
총 n번 파티션 × 각 O(n) = O(n^2)
```

**최적화 전략**:

```python
import random

def quick_sort_optimized(arr, low=0, high=None):
    """랜덤 피벗으로 최악의 경우 방지"""
    if high is None:
        high = len(arr) - 1
    if low < high:
        # 랜덤 피벗 선택
        rand_idx = random.randint(low, high)
        arr[rand_idx], arr[high] = arr[high], arr[rand_idx]

        pi = lomuto_partition(arr, low, high)
        quick_sort_optimized(arr, low, pi - 1)
        quick_sort_optimized(arr, pi + 1, high)
    return arr
```

| 항목 | 값 |
|------|-----|
| 시간복잡도 (평균) | O(n log n) |
| 시간복잡도 (최악) | O(n^2) - 랜덤 피벗으로 회피 가능 |
| 공간복잡도 | O(log n) - 재귀 스택 |
| 안정 정렬 | 아니오 |
| 제자리 정렬 | 예 (추가 배열 불필요) |

## 힙소트 (Heap Sort)

### 원리

힙소트는 **최대 힙(Max Heap)** 자료구조를 이용한 정렬이다.

1. 배열을 최대 힙으로 구성 (Build Max Heap)
2. 루트(최댓값)를 배열 끝으로 이동
3. 힙 크기를 줄이고 힙 속성 복원 (Heapify)
4. 반복

```
원본 배열: [4, 10, 3, 5, 1]

1단계: 최대 힙 구성
        10
       /  \
      5    3
     / \
    4   1
배열: [10, 5, 3, 4, 1]

2단계: 루트(10)를 끝으로 이동, 힙 크기 줄임
[1, 5, 3, 4 | 10]  → heapify → [5, 4, 3, 1 | 10]

3단계: 루트(5)를 끝으로 이동
[1, 4, 3 | 5, 10]  → heapify → [4, 1, 3 | 5, 10]

4단계: 루트(4)를 끝으로 이동
[3, 1 | 4, 5, 10]  → heapify → [3, 1 | 4, 5, 10]

5단계: 루트(3)를 끝으로 이동
[1 | 3, 4, 5, 10]  → 완료!
```

### Python 구현

```python
def heapify(arr, n, i):
    """
    i번째 노드를 루트로 하는 서브트리를 최대 힙으로 변환
    n: 힙의 크기
    """
    largest = i        # 루트
    left = 2 * i + 1   # 왼쪽 자식
    right = 2 * i + 2  # 오른쪽 자식

    # 왼쪽 자식이 루트보다 크면
    if left < n and arr[left] > arr[largest]:
        largest = left

    # 오른쪽 자식이 현재 최대보다 크면
    if right < n and arr[right] > arr[largest]:
        largest = right

    # 최대가 루트가 아니면 교환 후 재귀
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)


def heap_sort(arr):
    """힙소트 - O(n log n), 불안정, 제자리(in-place)"""
    n = len(arr)

    # 1단계: 최대 힙 구성 (마지막 내부 노드부터 역순)
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)

    # 2단계: 하나씩 루트를 끝으로 이동
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 루트(최대)를 끝으로
        heapify(arr, i, 0)  # 줄어든 힙에서 heapify

    return arr


# 테스트
arr = [4, 10, 3, 5, 1]
print(heap_sort(arr))  # [1, 3, 4, 5, 10]
```

| 항목 | 값 |
|------|-----|
| 시간복잡도 | 최선/평균/최악 모두 O(n log n) |
| 공간복잡도 | O(1) - 제자리 정렬 |
| 안정 정렬 | 아니오 |
| 특징 | 최악의 경우에도 O(n log n) 보장 |

## 카운팅 정렬 (Counting Sort)

### 원리

원소의 **개수를 세어서** 위치를 결정하는 비비교 기반 정렬이다. 원소의 범위 k가 작을 때 매우 효율적이다.

```
입력: [4, 2, 2, 8, 3, 3, 1]
범위: 0~8 (k=9)

1단계: 개수 세기 (count 배열)
인덱스: 0  1  2  3  4  5  6  7  8
개수:   0  1  2  2  1  0  0  0  1

2단계: 누적합 (시작 위치 결정)
인덱스: 0  1  2  3  4  5  6  7  8
누적:   0  1  3  5  6  6  6  6  7

3단계: 결과 배열 구성 (뒤에서부터 → 안정성 유지)
결과: [1, 2, 2, 3, 3, 4, 8]
```

### Python 구현

```python
def counting_sort(arr):
    """
    카운팅 정렬 - O(n + k), 안정 정렬
    k: 원소의 범위 (최댓값 - 최솟값 + 1)
    """
    if not arr:
        return arr

    max_val = max(arr)
    min_val = min(arr)
    k = max_val - min_val + 1  # 범위

    # 1단계: 개수 세기
    count = [0] * k
    for x in arr:
        count[x - min_val] += 1

    # 2단계: 누적합 계산
    for i in range(1, k):
        count[i] += count[i - 1]

    # 3단계: 결과 배열 구성 (뒤에서부터 순회 → 안정성)
    result = [0] * len(arr)
    for x in reversed(arr):
        count[x - min_val] -= 1
        result[count[x - min_val]] = x

    return result


# 테스트
arr = [4, 2, 2, 8, 3, 3, 1]
print(counting_sort(arr))  # [1, 2, 2, 3, 3, 4, 8]
```

| 항목 | 값 |
|------|-----|
| 시간복잡도 | O(n + k) |
| 공간복잡도 | O(n + k) |
| 안정 정렬 | 예 |
| 제한 | k가 매우 크면 비효율적 |

## 기수 정렬 (Radix Sort)

### 원리

각 **자릿수(digit)**별로 안정 정렬(보통 카운팅 정렬)을 반복 수행한다. LSD(Least Significant Digit)는 가장 낮은 자릿수부터, MSD(Most Significant Digit)는 가장 높은 자릿수부터 처리한다.

```
LSD 기수 정렬 과정 (10진법)

원본:   [170, 45, 75, 90, 802, 24, 2, 66]

1의 자리 정렬:
  [170, 90, 802, 2, 24, 45, 75, 66]

10의 자리 정렬:
  [802, 2, 24, 45, 66, 170, 75, 90]

100의 자리 정렬:
  [2, 24, 45, 66, 75, 90, 170, 802]
```

### LSD vs MSD

| 항목 | LSD (낮은 자릿수 먼저) | MSD (높은 자릿수 먼저) |
|------|----------------------|----------------------|
| 처리 순서 | 1의 자리 → 10의 자리 → ... | 최고 자릿수 → ... → 1의 자리 |
| 구현 | 단순 반복 | 재귀 (버킷별 분할) |
| 안정성 | 안정 (안정 정렬 사용 시) | 구현에 따라 다름 |
| 장점 | 구현 간단, 정수에 적합 | 문자열 정렬에 유리 |

### Python 구현 (LSD)

```python
def counting_sort_by_digit(arr, exp):
    """특정 자릿수(exp)를 기준으로 카운팅 정렬"""
    n = len(arr)
    output = [0] * n
    count = [0] * 10  # 0~9 자릿수

    # 해당 자릿수의 개수 세기
    for x in arr:
        digit = (x // exp) % 10
        count[digit] += 1

    # 누적합
    for i in range(1, 10):
        count[i] += count[i - 1]

    # 결과 배열 구성 (뒤에서부터 → 안정성)
    for x in reversed(arr):
        digit = (x // exp) % 10
        count[digit] -= 1
        output[count[digit]] = x

    # 원본 배열에 복사
    for i in range(n):
        arr[i] = output[i]


def radix_sort(arr):
    """
    기수 정렬 (LSD) - O(d * (n + k))
    d: 최대 자릿수, k: 기수(10진법이면 10)
    """
    if not arr:
        return arr

    max_val = max(arr)

    # 1의 자리부터 최대 자릿수까지 반복
    exp = 1
    while max_val // exp > 0:
        counting_sort_by_digit(arr, exp)
        exp *= 10

    return arr


# 테스트
arr = [170, 45, 75, 90, 802, 24, 2, 66]
print(radix_sort(arr))  # [2, 24, 45, 66, 75, 90, 170, 802]
```

| 항목 | 값 |
|------|-----|
| 시간복잡도 | O(d * (n + k)) |
| 공간복잡도 | O(n + k) |
| 안정 정렬 | 예 (LSD 방식) |
| 조건 | 양의 정수에 적합, 음수는 별도 처리 필요 |

## 안정성 (Stability)

### 안정 정렬이란?

**같은 키 값을 가진 원소들의 원래 상대적 순서가 정렬 후에도 유지**되는 것을 안정(stable) 정렬이라 한다.

```
안정 정렬 예시: 이름순으로 정렬된 데이터를 점수순으로 재정렬

원본 (이름순):
  ("Alice", 90), ("Bob", 80), ("Charlie", 90), ("Dave", 80)

안정 정렬 (점수순):
  ("Bob", 80), ("Dave", 80), ("Alice", 90), ("Charlie", 90)
  → 같은 점수(80, 90) 내에서 원래 이름순 유지

불안정 정렬 (점수순):
  ("Dave", 80), ("Bob", 80), ("Charlie", 90), ("Alice", 90)
  → 같은 점수 내 순서가 바뀔 수 있음
```

> 안정성이 중요한 경우: 다중 키 정렬(예: 1차 점수, 2차 이름), 데이터베이스 쿼리의 ORDER BY 등.

## 종합 비교 테이블

| 알고리즘 | 최선 | 평균 | 최악 | 공간 | 안정 | 비고 |
|----------|------|------|------|------|------|------|
| 버블 정렬 | O(n) | O(n^2) | O(n^2) | O(1) | 안정 | 교육용, 실무 사용 드묾 |
| 선택 정렬 | O(n^2) | O(n^2) | O(n^2) | O(1) | 불안정 | 교환 횟수 최소 |
| 삽입 정렬 | O(n) | O(n^2) | O(n^2) | O(1) | 안정 | 소규모/거의 정렬된 데이터에 우수 |
| 머지소트 | O(n log n) | O(n log n) | O(n log n) | O(n) | 안정 | 항상 O(n log n) 보장 |
| 퀵소트 | O(n log n) | O(n log n) | O(n^2) | O(log n) | 불안정 | 실무에서 가장 빠름 (캐시 친화적) |
| 힙소트 | O(n log n) | O(n log n) | O(n log n) | O(1) | 불안정 | 최악 O(n log n) + 제자리 |
| 카운팅 정렬 | O(n+k) | O(n+k) | O(n+k) | O(n+k) | 안정 | 범위 k가 작을 때 |
| 기수 정렬 | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 안정 | 자릿수 d가 작을 때 |

### 언제 어떤 알고리즘을 선택할까?

| 상황 | 추천 알고리즘 | 이유 |
|------|-------------|------|
| 범용 정렬 | 퀵소트 / 머지소트 | 평균 O(n log n) |
| 안정성 필요 | 머지소트 | 안정 + O(n log n) |
| 메모리 제한 | 힙소트 | O(1) 추가 공간 |
| 거의 정렬된 데이터 | 삽입 정렬 | 최선 O(n) |
| 정수 범위가 작음 | 카운팅/기수 정렬 | O(n) 수준 |
| Python 내장 | `sorted()` / `.sort()` | Tim Sort (머지+삽입 하이브리드) |

## 정리

| 항목 | 내용 |
|------|------|
| 비교 기반 하한 | O(n log n) - 비교만으로는 이보다 빠를 수 없음 |
| 머지소트 핵심 | 분할 → 재귀 정렬 → 병합, 항상 O(n log n), 안정 |
| 퀵소트 핵심 | 피벗 파티션 → 재귀, 평균 O(n log n), 최악 O(n^2) |
| 힙소트 핵심 | 최대 힙 구성 → 루트 추출 반복, O(n log n), 제자리 |
| 카운팅 정렬 | 개수 세기 → 위치 결정, O(n+k), 범위 제한 |
| 기수 정렬 | 자릿수별 안정 정렬 반복, O(d*(n+k)) |
| 안정성 | 같은 키의 상대 순서 유지 여부 |
| Python 기본 | Tim Sort - 머지소트 + 삽입 정렬 하이브리드, 안정 |
