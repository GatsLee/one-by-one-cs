## 핵심 개념

### 스택 (Stack) — LIFO

스택은 **후입선출(LIFO: Last In, First Out)** 구조다. 접시 쌓기를 떠올려보자. 접시를 쌓을 때 맨 위에 올리고, 꺼낼 때도 맨 위에서 꺼낸다. 중간에서 뺄 수 없다.

```
        push(40)                pop()
          │                       │
          v                       v
      ┌────────┐             ┌────────┐
      │   40   │  ← top      │   40   │ ← 이것이 나옴!
      ├────────┤             ├────────┤
      │   30   │             │   30   │ ← 새로운 top
      ├────────┤             ├────────┤
      │   20   │             │   20   │
      ├────────┤             ├────────┤
      │   10   │             │   10   │
      └────────┘             └────────┘
```

**스택의 핵심 연산:**

| 연산 | 설명 | 시간 복잡도 |
|------|------|------------|
| `push(item)` | 맨 위에 원소 추가 | O(1) |
| `pop()` | 맨 위 원소 꺼내기 (제거 + 반환) | O(1) |
| `peek()` / `top()` | 맨 위 원소 확인 (제거 안 함) | O(1) |
| `is_empty()` | 비어있는지 확인 | O(1) |
| `size()` | 원소 개수 | O(1) |

**실생활 비유:**
- 접시 쌓기: 마지막에 올린 접시를 먼저 꺼냄
- 웹 브라우저 뒤로가기: 방문 기록을 스택에 쌓고, 뒤로가기 누르면 pop
- Ctrl+Z (되돌리기): 작업을 스택에 쌓고, 되돌릴 때 pop
- 함수 호출 스택 (Call Stack): 함수가 호출되면 push, 반환되면 pop

### 큐 (Queue) — FIFO

큐는 **선입선출(FIFO: First In, First Out)** 구조다. 은행 줄서기를 떠올려보자. 먼저 온 사람이 먼저 서비스를 받는다.

```
 enqueue(40)                              dequeue()
     │                                       │
     v                                       v
   ┌────────┬────────┬────────┬────────┐
   │   10   │   20   │   30   │   40   │
   └────────┴────────┴────────┴────────┘
     ^                           ^
    front                       rear
   (먼저 나감)              (나중에 들어옴)
```

**큐의 핵심 연산:**

| 연산 | 설명 | 시간 복잡도 |
|------|------|------------|
| `enqueue(item)` | 뒤(rear)에 원소 추가 | O(1) |
| `dequeue()` | 앞(front)에서 원소 꺼내기 | O(1) |
| `peek()` / `front()` | 맨 앞 원소 확인 | O(1) |
| `is_empty()` | 비어있는지 확인 | O(1) |
| `size()` | 원소 개수 | O(1) |

**실생활 비유:**
- 은행 대기줄: 먼저 온 순서대로 처리
- 프린터 대기열: 먼저 요청한 문서부터 출력
- BFS(너비 우선 탐색): 탐색할 노드를 큐에 넣고 순서대로 방문
- 메시지 큐: 서버 간 비동기 통신 (RabbitMQ, Kafka 등)

### 콜 스택 (Call Stack)

프로그래밍에서 함수가 호출될 때마다 **콜 스택**에 프레임이 쌓인다. 이를 이해하면 재귀, 에러 추적(traceback) 등이 명확해진다.

```python
def a():
    b()      # b 호출
    print("a 끝")

def b():
    c()      # c 호출
    print("b 끝")

def c():
    print("c 실행")

a()  # 실행 시작
```

```
콜 스택 변화:

 a() 호출     b() 호출     c() 호출     c() 반환     b() 반환     a() 반환
┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│        │  │        │  │  c()   │  │        │  │        │  │        │
│        │  │  b()   │  │  b()   │  │  b()   │  │        │  │        │
│  a()   │  │  a()   │  │  a()   │  │  a()   │  │  a()   │  │        │
└────────┘  └────────┘  └────────┘  └────────┘  └────────┘  └────────┘

출력 순서: "c 실행" → "b 끝" → "a 끝"
```

Python의 기본 재귀 깊이 제한은 1000이다. 이를 넘으면 `RecursionError`가 발생한다. 이것은 콜 스택 오버플로를 방지하기 위함이다.

### 원형 큐 (Circular Queue)

일반 배열로 큐를 구현하면, dequeue할 때마다 앞쪽에 빈 공간이 생기는 문제가 있다. **원형 큐**는 배열의 끝이 처음과 연결된 것처럼 동작하여 이 공간을 재활용한다.

```
일반 배열 큐의 문제:
dequeue 반복 → [_, _, _, 40, 50]  ← 앞의 빈 공간 낭비!

원형 큐의 해결:
인덱스를 % (나머지) 연산으로 순환

         ┌───┐
      ┌──│ 0 │──┐
      │  └───┘  │
   ┌──┴─┐    ┌──┴─┐
   │ 4  │    │ 1  │
   └──┬─┘    └──┬─┘
      │  ┌───┐  │
      └──│ 3 │──┘
         │   │
         │ 2 │
         └───┘

front = 2, rear = 0 일 때:
인덱스 2, 3, 4, 0에 데이터 저장 (순환!)
다음 삽입 위치: (rear + 1) % size = 1
```

**핵심 공식:**
- 다음 위치: `(현재 인덱스 + 1) % 배열크기`
- 비어있는 조건: `front == rear`
- 가득 찬 조건: `(rear + 1) % 배열크기 == front`

### 덱 (Deque — Double-Ended Queue)

덱은 **양쪽 끝에서 삽입과 삭제가 모두 가능**한 자료구조다. 스택과 큐의 기능을 모두 가진 일반화된 버전이다.

```
  addFirst()                         addLast()
      │                                 │
      v                                 v
   ┌────────┬────────┬────────┬────────┐
   │   10   │   20   │   30   │   40   │
   └────────┴────────┴────────┴────────┘
      ^                                 ^
  removeFirst()                    removeLast()
```

Python의 `collections.deque`가 바로 이 자료구조다.

## 정리

| 항목 | 스택 (Stack) | 큐 (Queue) | 덱 (Deque) |
|------|-------------|-----------|------------|
| 원칙 | LIFO (후입선출) | FIFO (선입선출) | 양쪽 모두 가능 |
| 삽입 | push (top) | enqueue (rear) | 양쪽 끝 |
| 삭제 | pop (top) | dequeue (front) | 양쪽 끝 |
| Python 구현 | list | collections.deque | collections.deque |
| 시간 복잡도 | 모든 연산 O(1) | 모든 연산 O(1) | 모든 연산 O(1) |
| 대표 활용 | 괄호 검사, 되돌리기, DFS | BFS, 작업 대기열 | 슬라이딩 윈도우 |

**주의사항:**
- 큐를 Python list로 구현하면 `pop(0)`이 O(n)이므로 반드시 `deque`를 사용할 것
- 스택은 Python list의 `append()`/`pop()`으로 충분 (둘 다 O(1))
- 원형 큐는 고정 크기 버퍼가 필요할 때 사용 (OS의 키보드 입력 버퍼 등)
