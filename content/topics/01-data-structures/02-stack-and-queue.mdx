## 핵심 개념

### 스택 (Stack) — LIFO

스택은 **후입선출(LIFO: Last In, First Out)** 구조다. 접시 쌓기를 떠올려보자. 접시를 쌓을 때 맨 위에 올리고, 꺼낼 때도 맨 위에서 꺼낸다. 중간에서 뺄 수 없다.

```
        push(40)                pop()
          │                       │
          v                       v
      ┌────────┐             ┌────────┐
      │   40   │  ← top      │   40   │ ← 이것이 나옴!
      ├────────┤             ├────────┤
      │   30   │             │   30   │ ← 새로운 top
      ├────────┤             ├────────┤
      │   20   │             │   20   │
      ├────────┤             ├────────┤
      │   10   │             │   10   │
      └────────┘             └────────┘
```

**스택의 핵심 연산:**

| 연산 | 설명 | 시간 복잡도 |
|------|------|------------|
| `push(item)` | 맨 위에 원소 추가 | O(1) |
| `pop()` | 맨 위 원소 꺼내기 (제거 + 반환) | O(1) |
| `peek()` / `top()` | 맨 위 원소 확인 (제거 안 함) | O(1) |
| `is_empty()` | 비어있는지 확인 | O(1) |
| `size()` | 원소 개수 | O(1) |

**실생활 비유:**
- 접시 쌓기: 마지막에 올린 접시를 먼저 꺼냄
- 웹 브라우저 뒤로가기: 방문 기록을 스택에 쌓고, 뒤로가기 누르면 pop
- Ctrl+Z (되돌리기): 작업을 스택에 쌓고, 되돌릴 때 pop
- 함수 호출 스택 (Call Stack): 함수가 호출되면 push, 반환되면 pop

### 큐 (Queue) — FIFO

큐는 **선입선출(FIFO: First In, First Out)** 구조다. 은행 줄서기를 떠올려보자. 먼저 온 사람이 먼저 서비스를 받는다.

```
 enqueue(40)                              dequeue()
     │                                       │
     v                                       v
   ┌────────┬────────┬────────┬────────┐
   │   10   │   20   │   30   │   40   │
   └────────┴────────┴────────┴────────┘
     ^                           ^
    front                       rear
   (먼저 나감)              (나중에 들어옴)
```

**큐의 핵심 연산:**

| 연산 | 설명 | 시간 복잡도 |
|------|------|------------|
| `enqueue(item)` | 뒤(rear)에 원소 추가 | O(1) |
| `dequeue()` | 앞(front)에서 원소 꺼내기 | O(1) |
| `peek()` / `front()` | 맨 앞 원소 확인 | O(1) |
| `is_empty()` | 비어있는지 확인 | O(1) |
| `size()` | 원소 개수 | O(1) |

**실생활 비유:**
- 은행 대기줄: 먼저 온 순서대로 처리
- 프린터 대기열: 먼저 요청한 문서부터 출력
- BFS(너비 우선 탐색): 탐색할 노드를 큐에 넣고 순서대로 방문
- 메시지 큐: 서버 간 비동기 통신 (RabbitMQ, Kafka 등)

### 콜 스택 (Call Stack)

프로그래밍에서 함수가 호출될 때마다 **콜 스택**에 프레임이 쌓인다. 이를 이해하면 재귀, 에러 추적(traceback) 등이 명확해진다.

```python
def a():
    b()      # b 호출
    print("a 끝")

def b():
    c()      # c 호출
    print("b 끝")

def c():
    print("c 실행")

a()  # 실행 시작
```

```
콜 스택 변화:

 a() 호출     b() 호출     c() 호출     c() 반환     b() 반환     a() 반환
┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐  ┌────────┐
│        │  │        │  │  c()   │  │        │  │        │  │        │
│        │  │  b()   │  │  b()   │  │  b()   │  │        │  │        │
│  a()   │  │  a()   │  │  a()   │  │  a()   │  │  a()   │  │        │
└────────┘  └────────┘  └────────┘  └────────┘  └────────┘  └────────┘

출력 순서: "c 실행" → "b 끝" → "a 끝"
```

Python의 기본 재귀 깊이 제한은 1000이다. 이를 넘으면 `RecursionError`가 발생한다. 이것은 콜 스택 오버플로를 방지하기 위함이다.

### 원형 큐 (Circular Queue)

일반 배열로 큐를 구현하면, dequeue할 때마다 앞쪽에 빈 공간이 생기는 문제가 있다. **원형 큐**는 배열의 끝이 처음과 연결된 것처럼 동작하여 이 공간을 재활용한다.

```
일반 배열 큐의 문제:
dequeue 반복 → [_, _, _, 40, 50]  ← 앞의 빈 공간 낭비!

원형 큐의 해결:
인덱스를 % (나머지) 연산으로 순환

         ┌───┐
      ┌──│ 0 │──┐
      │  └───┘  │
   ┌──┴─┐    ┌──┴─┐
   │ 4  │    │ 1  │
   └──┬─┘    └──┬─┘
      │  ┌───┐  │
      └──│ 3 │──┘
         │   │
         │ 2 │
         └───┘

front = 2, rear = 0 일 때:
인덱스 2, 3, 4, 0에 데이터 저장 (순환!)
다음 삽입 위치: (rear + 1) % size = 1
```

**핵심 공식:**
- 다음 위치: `(현재 인덱스 + 1) % 배열크기`
- 비어있는 조건: `front == rear`
- 가득 찬 조건: `(rear + 1) % 배열크기 == front`

### 덱 (Deque — Double-Ended Queue)

덱은 **양쪽 끝에서 삽입과 삭제가 모두 가능**한 자료구조다. 스택과 큐의 기능을 모두 가진 일반화된 버전이다.

```
  addFirst()                         addLast()
      │                                 │
      v                                 v
   ┌────────┬────────┬────────┬────────┐
   │   10   │   20   │   30   │   40   │
   └────────┴────────┴────────┴────────┘
      ^                                 ^
  removeFirst()                    removeLast()
```

Python의 `collections.deque`가 바로 이 자료구조다.

## 코드 예시

### 1. 스택 구현 (Python list 기반)

```python
class Stack:
    """Python list를 이용한 스택 구현"""
    def __init__(self):
        self._items = []

    def push(self, item):
        """맨 위에 원소 추가 - O(1) 분할상환"""
        self._items.append(item)

    def pop(self):
        """맨 위 원소 꺼내기 - O(1)"""
        if self.is_empty():
            raise IndexError("빈 스택에서 pop 할 수 없습니다")
        return self._items.pop()

    def peek(self):
        """맨 위 원소 확인 (제거 안 함) - O(1)"""
        if self.is_empty():
            raise IndexError("빈 스택입니다")
        return self._items[-1]

    def is_empty(self):
        """비어있는지 확인 - O(1)"""
        return len(self._items) == 0

    def size(self):
        """원소 개수 - O(1)"""
        return len(self._items)

    def __repr__(self):
        return f"Stack(top → {self._items[::-1]})"


# === 사용 예시 ===
s = Stack()
s.push(10)
s.push(20)
s.push(30)
print(s)          # Stack(top → [30, 20, 10])
print(s.peek())   # 30
print(s.pop())    # 30
print(s)          # Stack(top → [20, 10])
```

### 2. 스택 활용: 괄호 유효성 검사

```python
def is_valid_parentheses(s: str) -> bool:
    """
    괄호 문자열이 유효한지 검사한다.

    유효한 예: "()", "()[]{}", "{[]}"
    유효하지 않은 예: "(]", "([)]", "((("

    알고리즘:
    1. 여는 괄호를 만나면 스택에 push
    2. 닫는 괄호를 만나면 스택에서 pop하여 짝이 맞는지 확인
    3. 끝까지 검사 후 스택이 비어있으면 유효
    """
    stack = []
    # 닫는 괄호 → 대응하는 여는 괄호 매핑
    matching = {')': '(', ']': '[', '}': '{'}

    for char in s:
        if char in '([{':
            # 여는 괄호: 스택에 push
            stack.append(char)
        elif char in ')]}':
            # 닫는 괄호: 스택이 비어있거나 짝이 안 맞으면 실패
            if not stack or stack[-1] != matching[char]:
                return False
            stack.pop()

    # 모든 괄호가 짝지어졌으면 스택이 비어있어야 함
    return len(stack) == 0


# 테스트
print(is_valid_parentheses("()[]{}"))   # True
print(is_valid_parentheses("{[()]}"))   # True
print(is_valid_parentheses("(]"))       # False
print(is_valid_parentheses("(("))       # False
print(is_valid_parentheses(""))         # True (빈 문자열은 유효)
```

### 3. 큐 구현 (collections.deque 기반)

```python
from collections import deque

class Queue:
    """collections.deque를 이용한 큐 구현"""
    def __init__(self):
        self._items = deque()

    def enqueue(self, item):
        """뒤에 원소 추가 - O(1)"""
        self._items.append(item)

    def dequeue(self):
        """앞에서 원소 꺼내기 - O(1)"""
        if self.is_empty():
            raise IndexError("빈 큐에서 dequeue 할 수 없습니다")
        return self._items.popleft()

    def peek(self):
        """맨 앞 원소 확인 - O(1)"""
        if self.is_empty():
            raise IndexError("빈 큐입니다")
        return self._items[0]

    def is_empty(self):
        return len(self._items) == 0

    def size(self):
        return len(self._items)

    def __repr__(self):
        return f"Queue(front → {list(self._items)})"


# === 사용 예시 ===
q = Queue()
q.enqueue("작업1")
q.enqueue("작업2")
q.enqueue("작업3")
print(q)            # Queue(front → ['작업1', '작업2', '작업3'])
print(q.dequeue())  # 작업1
print(q.dequeue())  # 작업2
print(q)            # Queue(front → ['작업3'])
```

### 4. 큐를 list로 구현하면 안 되는 이유

```python
import time
from collections import deque

n = 100_000

# list로 큐 구현 (느림!)
lst_queue = []
start = time.perf_counter()
for i in range(n):
    lst_queue.append(i)
for i in range(n):
    lst_queue.pop(0)  # O(n) — 매번 모든 원소를 앞으로 이동!
list_time = time.perf_counter() - start

# deque로 큐 구현 (빠름!)
deque_queue = deque()
start = time.perf_counter()
for i in range(n):
    deque_queue.append(i)
for i in range(n):
    deque_queue.popleft()  # O(1)
deque_time = time.perf_counter() - start

print(f"list  큐: {list_time:.4f}초")
print(f"deque 큐: {deque_time:.4f}초")
# list는 deque보다 수십~수백 배 느림
```

### 5. 원형 큐 구현

```python
class CircularQueue:
    """고정 크기 원형 큐"""
    def __init__(self, capacity):
        self._capacity = capacity + 1  # 빈 칸 1개를 구분용으로 사용
        self._data = [None] * self._capacity
        self._front = 0
        self._rear = 0

    def enqueue(self, item):
        """뒤에 원소 추가"""
        if self.is_full():
            raise OverflowError("큐가 가득 찼습니다")
        self._data[self._rear] = item
        self._rear = (self._rear + 1) % self._capacity  # 순환!

    def dequeue(self):
        """앞에서 원소 꺼내기"""
        if self.is_empty():
            raise IndexError("빈 큐입니다")
        item = self._data[self._front]
        self._data[self._front] = None  # 정리 (선택 사항)
        self._front = (self._front + 1) % self._capacity  # 순환!
        return item

    def is_empty(self):
        return self._front == self._rear

    def is_full(self):
        return (self._rear + 1) % self._capacity == self._front

    def size(self):
        return (self._rear - self._front) % self._capacity

    def __repr__(self):
        if self.is_empty():
            return "CircularQueue(빈 큐)"
        items = []
        i = self._front
        while i != self._rear:
            items.append(str(self._data[i]))
            i = (i + 1) % self._capacity
        return f"CircularQueue(front → [{', '.join(items)}])"


# === 사용 예시 ===
cq = CircularQueue(4)  # 용량 4
cq.enqueue(10)
cq.enqueue(20)
cq.enqueue(30)
cq.enqueue(40)
print(cq)  # CircularQueue(front → [10, 20, 30, 40])

cq.dequeue()  # 10 제거
cq.dequeue()  # 20 제거
cq.enqueue(50)  # 앞쪽 빈 공간을 재활용!
cq.enqueue(60)
print(cq)  # CircularQueue(front → [30, 40, 50, 60])
```

### 6. 스택으로 후위 표기법 계산

```python
def evaluate_postfix(expression: str) -> float:
    """
    후위 표기법(Reverse Polish Notation) 수식을 계산한다.

    후위 표기법: 연산자가 피연산자 뒤에 온다
    예) "3 4 +" → 3 + 4 = 7
        "3 4 + 2 *" → (3 + 4) * 2 = 14

    알고리즘:
    1. 숫자를 만나면 스택에 push
    2. 연산자를 만나면 스택에서 2개 pop 하여 계산, 결과를 push
    3. 마지막에 스택에 남은 값이 결과
    """
    stack = []
    operators = {'+', '-', '*', '/'}

    for token in expression.split():
        if token in operators:
            b = stack.pop()  # 두 번째 피연산자 (먼저 pop한 것이 오른쪽)
            a = stack.pop()  # 첫 번째 피연산자
            if token == '+':
                stack.append(a + b)
            elif token == '-':
                stack.append(a - b)
            elif token == '*':
                stack.append(a * b)
            elif token == '/':
                stack.append(a / b)
        else:
            stack.append(float(token))

    return stack[0]


# 테스트
print(evaluate_postfix("3 4 +"))        # 7.0  (3 + 4)
print(evaluate_postfix("3 4 + 2 *"))    # 14.0 ((3 + 4) * 2)
print(evaluate_postfix("5 1 2 + 4 * + 3 -"))  # 14.0 (5 + ((1+2)*4) - 3)
```

### 7. BFS에서 큐 활용

```python
from collections import deque

def bfs(graph, start):
    """
    너비 우선 탐색 (BFS)

    큐를 사용하여 가까운 노드부터 차례대로 방문한다.

    graph: 인접 리스트 (딕셔너리)
    start: 시작 노드
    """
    visited = set()       # 방문한 노드 기록
    queue = deque([start])  # 탐색할 노드를 큐에 저장
    visited.add(start)
    order = []            # 방문 순서 기록

    while queue:
        node = queue.popleft()  # 큐에서 꺼내기 (FIFO)
        order.append(node)

        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)  # 이웃을 큐에 추가

    return order


# 그래프 예시
#   A --- B --- D
#   |         |
#   C --- E ---
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D', 'E'],
    'C': ['A', 'E'],
    'D': ['B'],
    'E': ['B', 'C']
}

print(bfs(graph, 'A'))  # ['A', 'B', 'C', 'D', 'E']
# A에서 가까운 순서대로 방문
```

## 정리

| 항목 | 스택 (Stack) | 큐 (Queue) | 덱 (Deque) |
|------|-------------|-----------|------------|
| 원칙 | LIFO (후입선출) | FIFO (선입선출) | 양쪽 모두 가능 |
| 삽입 | push (top) | enqueue (rear) | 양쪽 끝 |
| 삭제 | pop (top) | dequeue (front) | 양쪽 끝 |
| Python 구현 | list | collections.deque | collections.deque |
| 시간 복잡도 | 모든 연산 O(1) | 모든 연산 O(1) | 모든 연산 O(1) |
| 대표 활용 | 괄호 검사, 되돌리기, DFS | BFS, 작업 대기열 | 슬라이딩 윈도우 |

**주의사항:**
- 큐를 Python list로 구현하면 `pop(0)`이 O(n)이므로 반드시 `deque`를 사용할 것
- 스택은 Python list의 `append()`/`pop()`으로 충분 (둘 다 O(1))
- 원형 큐는 고정 크기 버퍼가 필요할 때 사용 (OS의 키보드 입력 버퍼 등)
