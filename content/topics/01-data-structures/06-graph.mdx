## 핵심 개념

### 그래프(Graph)란?

그래프는 **정점(Vertex)**과 **간선(Edge)**으로 이루어진 자료구조로, 개체 간의 관계를 표현한다. 트리도 그래프의 특수한 형태(사이클이 없는 연결 그래프)다.

> **비유**: 소셜 네트워크에서 사람 = 정점, 친구 관계 = 간선. 도시 간 도로망에서 도시 = 정점, 도로 = 간선.

### 그래프 용어

```
    A ---5--- B
    |       / |
    3     2   4
    |   /     |
    C ---1--- D
```

| 용어 | 설명 | 예시 |
|------|------|------|
| **정점(Vertex/Node)** | 그래프의 개체 | A, B, C, D |
| **간선(Edge)** | 정점 간 연결 | A-B, A-C, B-C, B-D, C-D |
| **인접(Adjacent)** | 간선으로 직접 연결됨 | A와 B는 인접 |
| **차수(Degree)** | 정점에 연결된 간선 수 | A의 차수 = 2 |
| **경로(Path)** | 정점들의 연결된 순서 | A→C→D |
| **가중치(Weight)** | 간선에 부여된 값/비용 | A-B 간선의 가중치 = 5 |

### 그래프의 종류

```
[무방향 그래프]        [방향 그래프(Digraph)]    [가중치 그래프]
  A --- B              A → B                   A --5-- B
  |     |              ↑   ↓                   |       |
  C --- D              C ← D                   3       4
                                               |       |
                                               C --1-- D
```

| 종류 | 설명 |
|------|------|
| **무방향 그래프** | 간선에 방향이 없음 (A-B = B-A) |
| **방향 그래프** | 간선에 방향이 있음 (A→B ≠ B→A) |
| **가중치 그래프** | 간선에 비용/거리 등의 가중치가 있음 |
| **연결 그래프** | 모든 정점 간에 경로가 존재 |
| **비연결 그래프** | 경로가 없는 정점 쌍이 존재 |
| **완전 그래프** | 모든 정점 쌍이 간선으로 연결 |
| **DAG** | 방향 비순환 그래프 (Directed Acyclic Graph) |

**방향 그래프의 차수:**
- **진입 차수(In-degree)**: 해당 정점으로 들어오는 간선 수
- **진출 차수(Out-degree)**: 해당 정점에서 나가는 간선 수

### 그래프 표현 방법

#### 1. 인접 행렬 (Adjacency Matrix)

정점 수 V에 대해 V x V 크기의 2차원 배열. `matrix[i][j] = 1`이면 i에서 j로 간선 존재.

```
그래프:                  인접 행렬:
  0 --- 1                    0  1  2  3
  |     |                0 [ 0, 1, 1, 0 ]
  2 --- 3                1 [ 1, 0, 0, 1 ]
                         2 [ 1, 0, 0, 1 ]
                         3 [ 0, 1, 1, 0 ]
```

| 장점 | 단점 |
|------|------|
| 간선 존재 여부 O(1) 확인 | 공간 O(V^2) — 희소 그래프에 비효율 |
| 간선 추가/삭제 O(1) | 모든 이웃 탐색 O(V) |
| 구현이 단순 | 정점 수가 많으면 메모리 낭비 |

#### 2. 인접 리스트 (Adjacency List)

각 정점에 대해 연결된 정점들의 리스트를 저장.

```
그래프:                  인접 리스트:
  0 --- 1                0: [1, 2]
  |     |                1: [0, 3]
  2 --- 3                2: [0, 3]
                         3: [1, 2]
```

| 장점 | 단점 |
|------|------|
| 공간 O(V + E) — 효율적 | 간선 존재 여부 O(degree) |
| 이웃 탐색이 빠름 | 인접 행렬보다 약간 복잡한 구현 |
| 대부분의 실제 그래프에 적합 | |

**언제 어떤 것을 사용할까?**
- **밀집 그래프** (간선이 많음, E ≈ V^2): 인접 행렬
- **희소 그래프** (간선이 적음, E &lt;&lt; V²): 인접 리스트 (대부분의 경우)

### 연결 요소 (Connected Component)

무방향 그래프에서 서로 연결된 정점들의 최대 집합.

```
  0 --- 1     3 --- 4     6
        |           |
        2           5

연결 요소 1: {0, 1, 2}
연결 요소 2: {3, 4, 5}
연결 요소 3: {6}
```

DFS 또는 BFS로 각 연결 요소를 찾을 수 있다.

### 사이클 검출

- **무방향 그래프**: DFS에서 이미 방문한 정점을 다시 만나면 사이클 (단, 부모 제외)
- **방향 그래프**: DFS에서 현재 재귀 스택에 있는 정점을 다시 만나면 사이클

### 최단 경로 알고리즘

#### 다익스트라 (Dijkstra)

**양의 가중치** 그래프에서 단일 출발점 최단 경로를 구하는 알고리즘.

**핵심 아이디어**: 현재까지 가장 가까운 미방문 정점을 선택하고, 그 정점을 통해 이웃의 거리를 갱신 (탐욕적 접근).

```
시작 정점: A

        A ---1--- B ---3--- D
        |         |         |
        4         2         1
        |         |         |
        C ---5--- E ---2--- F

단계별 최단 거리:
Step 0: A=0, B=inf, C=inf, D=inf, E=inf, F=inf
Step 1: A 방문 → B=1, C=4
Step 2: B 방문 → D=4, E=3
Step 3: E 방문 → F=5, C=4(갱신 안됨)
Step 4: C 방문 → (갱신 없음)
Step 5: D 방문 → F=5(갱신 안됨)
Step 6: F 방문 → 완료

최종: A=0, B=1, C=4, D=4, E=3, F=5
```

시간 복잡도: 우선순위 큐 사용 시 **O((V + E) log V)**

#### 벨만-포드 (Bellman-Ford)

**음의 가중치**가 있는 그래프에서도 사용 가능한 최단 경로 알고리즘.

- 모든 간선에 대해 완화(relaxation)를 V-1번 반복
- V번째 반복에서도 갱신이 일어나면 음의 사이클 존재
- 시간 복잡도: **O(V * E)**

| 비교 | 다익스트라 | 벨만-포드 |
|------|-----------|-----------|
| 음의 가중치 | 불가 | 가능 |
| 음의 사이클 검출 | 불가 | 가능 |
| 시간 복잡도 | O((V+E) log V) | O(V * E) |
| 사용 자료구조 | 우선순위 큐 | 간선 리스트 |

## 코드 예시

### 인접 리스트 및 인접 행렬 구현

```python
from collections import defaultdict

class GraphAdjList:
    """인접 리스트 기반 그래프 (무방향)"""

    def __init__(self):
        self.graph = defaultdict(list)

    def add_edge(self, u, v):
        """무방향 간선 추가"""
        self.graph[u].append(v)
        self.graph[v].append(u)

    def neighbors(self, v):
        """정점 v의 이웃 반환"""
        return self.graph[v]

    def has_edge(self, u, v):
        """간선 존재 여부 — O(degree)"""
        return v in self.graph[u]

    def display(self):
        for vertex in sorted(self.graph):
            print(f"{vertex}: {self.graph[vertex]}")


class GraphAdjMatrix:
    """인접 행렬 기반 그래프 (무방향)"""

    def __init__(self, num_vertices):
        self.V = num_vertices
        self.matrix = [[0] * num_vertices for _ in range(num_vertices)]

    def add_edge(self, u, v):
        """무방향 간선 추가"""
        self.matrix[u][v] = 1
        self.matrix[v][u] = 1

    def has_edge(self, u, v):
        """간선 존재 여부 — O(1)"""
        return self.matrix[u][v] == 1

    def neighbors(self, v):
        """정점 v의 이웃 반환"""
        return [i for i in range(self.V) if self.matrix[v][i] == 1]

    def display(self):
        for row in self.matrix:
            print(row)


# === 사용 예시 ===
g = GraphAdjList()
g.add_edge(0, 1)
g.add_edge(0, 2)
g.add_edge(1, 3)
g.add_edge(2, 3)
g.display()
# 0: [1, 2]
# 1: [0, 3]
# 2: [0, 3]
# 3: [1, 2]
```

### 연결 요소 찾기

```python
def find_connected_components(graph, num_vertices):
    """
    DFS로 연결 요소 찾기.
    graph: 인접 리스트 (dict)
    반환: 각 연결 요소를 리스트로 묶은 리스트
    """
    visited = set()
    components = []

    def dfs(v, component):
        visited.add(v)
        component.append(v)
        for neighbor in graph.get(v, []):
            if neighbor not in visited:
                dfs(neighbor, component)

    for v in range(num_vertices):
        if v not in visited:
            component = []
            dfs(v, component)
            components.append(component)

    return components


# === 예시 ===
graph = {
    0: [1], 1: [0, 2], 2: [1],      # 연결 요소 1
    3: [4], 4: [3, 5], 5: [4],      # 연결 요소 2
    6: []                            # 연결 요소 3
}
print(find_connected_components(graph, 7))
# [[0, 1, 2], [3, 4, 5], [6]]
```

### 사이클 검출 (방향 그래프)

```python
def has_cycle_directed(graph, num_vertices):
    """
    방향 그래프에서 사이클 검출 (DFS 기반).
    white(0): 미방문, gray(1): 현재 경로상, black(2): 탐색 완료
    """
    color = [0] * num_vertices  # 0=white, 1=gray, 2=black

    def dfs(v):
        color[v] = 1  # gray: 현재 탐색 중
        for neighbor in graph.get(v, []):
            if color[neighbor] == 1:    # gray 노드를 만남 → 사이클!
                return True
            if color[neighbor] == 0:    # white 노드 → 탐색 계속
                if dfs(neighbor):
                    return True
        color[v] = 2  # black: 탐색 완료
        return False

    for v in range(num_vertices):
        if color[v] == 0:
            if dfs(v):
                return True
    return False


# 사이클 있는 그래프: 0→1→2→0
graph_cycle = {0: [1], 1: [2], 2: [0]}
print(has_cycle_directed(graph_cycle, 3))     # True

# 사이클 없는 그래프: 0→1→2
graph_no_cycle = {0: [1], 1: [2], 2: []}
print(has_cycle_directed(graph_no_cycle, 3))  # False
```

### 다익스트라 알고리즘

```python
import heapq

def dijkstra(graph, start):
    """
    다익스트라 최단 경로 알고리즘.
    graph: {정점: [(이웃, 가중치), ...]}
    반환: {정점: 최단 거리}
    """
    # 모든 정점의 초기 거리를 무한대로 설정
    dist = {v: float('inf') for v in graph}
    dist[start] = 0

    # 우선순위 큐: (거리, 정점)
    pq = [(0, start)]

    while pq:
        d, u = heapq.heappop(pq)

        # 이미 더 짧은 경로를 찾았으면 스킵
        if d > dist[u]:
            continue

        # 이웃 정점들의 거리 갱신
        for v, weight in graph[u]:
            new_dist = dist[u] + weight
            if new_dist < dist[v]:           # 더 짧은 경로 발견
                dist[v] = new_dist
                heapq.heappush(pq, (new_dist, v))

    return dist


# === 예시 ===
graph = {
    'A': [('B', 1), ('C', 4)],
    'B': [('A', 1), ('D', 3), ('E', 2)],
    'C': [('A', 4), ('E', 5)],
    'D': [('B', 3), ('F', 1)],
    'E': [('B', 2), ('F', 2), ('C', 5)],
    'F': [('D', 1), ('E', 2)]
}

distances = dijkstra(graph, 'A')
for v, d in sorted(distances.items()):
    print(f"A → {v}: {d}")
# A → A: 0
# A → B: 1
# A → C: 4
# A → D: 4
# A → E: 3
# A → F: 5
```

### 벨만-포드 알고리즘

```python
def bellman_ford(vertices, edges, start):
    """
    벨만-포드 최단 경로 알고리즘.
    vertices: 정점 리스트
    edges: [(출발, 도착, 가중치), ...]
    반환: (거리 딕셔너리, 음의 사이클 존재 여부)
    """
    dist = {v: float('inf') for v in vertices}
    dist[start] = 0

    # V-1번 모든 간선에 대해 완화(relaxation)
    for _ in range(len(vertices) - 1):
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # V번째 반복에서 갱신이 일어나면 음의 사이클 존재
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return dist, True  # 음의 사이클 있음

    return dist, False


# === 예시 (음의 가중치 포함) ===
vertices = ['A', 'B', 'C', 'D']
edges = [
    ('A', 'B', 1),
    ('A', 'C', 4),
    ('B', 'C', -2),    # 음의 가중치
    ('B', 'D', 3),
    ('C', 'D', 1)
]

dist, has_neg_cycle = bellman_ford(vertices, edges, 'A')
print(f"음의 사이클: {has_neg_cycle}")     # False
for v, d in sorted(dist.items()):
    print(f"A → {v}: {d}")
# A → A: 0
# A → B: 1
# A → C: -1
# A → D: 0
```

## 정리

| 항목 | 내용 |
|------|------|
| 그래프 | 정점(V)과 간선(E)으로 이루어진 자료구조 |
| 방향 그래프 | 간선에 방향이 있음 (A→B ≠ B→A) |
| 가중치 그래프 | 간선에 비용/거리 등의 값이 있음 |
| 인접 행렬 | V x V 배열, 간선 확인 O(1), 공간 O(V^2) |
| 인접 리스트 | 각 정점별 이웃 리스트, 공간 O(V+E) |
| 연결 요소 | 서로 연결된 정점들의 최대 집합 |
| 사이클 검출 | 방향 그래프: DFS에서 gray 노드 재방문 시 사이클 |
| 다익스트라 | 양의 가중치, 우선순위 큐, O((V+E) log V) |
| 벨만-포드 | 음의 가중치 가능, O(V*E), 음의 사이클 검출 가능 |
