## 핵심 개념

### CPU 스케줄링이란?

여러 프로세스가 CPU를 사용하기 위해 경쟁할 때, **어떤 프로세스에 CPU를 할당할지 결정**하는 것이 CPU 스케줄링이다.

> **비유**: 은행 창구에 여러 고객이 대기 중일 때, "누구를 먼저 처리할 것인가?"를 결정하는 것.

### 스케줄링 목표

| 목표 | 설명 |
|------|------|
| **CPU 이용률 (Utilization)** | CPU가 놀지 않고 일하는 비율 (높을수록 좋음) |
| **처리율 (Throughput)** | 단위 시간당 완료되는 프로세스 수 (높을수록 좋음) |
| **대기 시간 (Waiting Time)** | 준비 큐에서 대기한 총 시간 (낮을수록 좋음) |
| **응답 시간 (Response Time)** | 요청 후 첫 응답까지의 시간 (낮을수록 좋음) |
| **반환 시간 (Turnaround Time)** | 프로세스 도착~완료까지 총 시간 (낮을수록 좋음) |

```
반환 시간 = 대기 시간 + 실행 시간 (버스트 타임)
대기 시간 = 반환 시간 - 실행 시간
```

### 선점 vs 비선점

| 구분 | 비선점 (Non-preemptive) | 선점 (Preemptive) |
|------|----------------------|------------------|
| 동작 | 프로세스가 자발적으로 CPU 반납 | OS가 강제로 CPU 회수 가능 |
| 장점 | 구현 단순, 컨텍스트 스위칭 적음 | 응답 시간 짧음, 공정성 |
| 단점 | 긴 프로세스가 CPU 독점 가능 | 컨텍스트 스위칭 오버헤드 |
| 예시 | FCFS, SJF(비선점) | Round Robin, SRTF |

### 1. FCFS (First Come First Served)

**먼저 온 프로세스를 먼저 처리**. 가장 단순한 비선점 스케줄링.

```
도착 순서: P1(24ms) → P2(3ms) → P3(3ms)

간트 차트:
|--- P1 (24) ---|P2(3)|P3(3)|
0              24    27    30

대기 시간: P1=0, P2=24, P3=27
평균 대기 시간: (0 + 24 + 27) / 3 = 17ms
```

**호위 효과(Convoy Effect)**: 긴 프로세스 뒤에 짧은 프로세스들이 오래 대기하는 현상.

```
순서를 바꾸면? P2(3ms) → P3(3ms) → P1(24ms)

|P2(3)|P3(3)|--- P1 (24) ---|
0     3     6              30

대기 시간: P2=0, P3=3, P1=6
평균 대기 시간: (0 + 3 + 6) / 3 = 3ms ← 훨씬 짧음!
```

### 2. SJF (Shortest Job First)

**실행 시간(버스트 타임)이 가장 짧은 프로세스를 먼저 실행**. 비선점 방식에서 평균 대기 시간이 이론적으로 최소.

```
프로세스: P1(6ms), P2(8ms), P3(7ms), P4(3ms)
모두 시간 0에 도착

SJF 순서: P4(3) → P1(6) → P3(7) → P2(8)

|P4(3)|-- P1(6) --|-- P3(7) --|-- P2(8) --|
0     3           9          16          24

대기 시간: P4=0, P1=3, P3=9, P2=16
평균 대기 시간: (0 + 3 + 9 + 16) / 4 = 7ms
```

**SRTF (Shortest Remaining Time First)**: SJF의 선점 버전. 남은 실행 시간이 더 짧은 프로세스가 도착하면 현재 프로세스를 중단하고 교체.

```
프로세스:     도착시간  버스트
P1           0        8
P2           1        4
P3           2        9
P4           3        5

SRTF:
|P1|-- P2(4) --|-- P4(5) --|P1(7남음)|-- P3(9) --|
0  1           5          10        17          26

대기 시간:
P1: (10-1) = 9 (1에서 선점, 10에서 재개)
P2: 0
P3: (17-2) = 15
P4: (5-3) = 2
평균: (9 + 0 + 15 + 2) / 4 = 6.5ms
```

**SJF의 문제**: 실행 시간을 미리 알 수 없다. 과거 데이터로 **예측**해야 한다.

### 3. Round Robin (RR)

**시간 할당량(Time Quantum, q)**을 정해서 각 프로세스가 그만큼만 실행 후 다음 프로세스로 교체. 선점 방식.

```
프로세스: P1(24ms), P2(3ms), P3(3ms)
시간 할당량 q = 4ms

|P1(4)|P2(3)|P3(3)|P1(4)|P1(4)|P1(4)|P1(4)|P1(4)|
0     4     7    10    14    18    22    26    30

대기 시간: P1 = (10-4) = 6 (총 대기), P2 = 4, P3 = 7
          → P1: 0~4 실행, 4~10 대기, 10~30 실행
          → P1 대기시간 = 30 - 24 - 0 = 6
평균 대기 시간: (6 + 4 + 7) / 3 = 5.67ms
```

**시간 할당량(q)의 선택:**
- q가 너무 크면 → FCFS와 같아짐
- q가 너무 작으면 → 컨텍스트 스위칭 오버헤드가 커짐
- 일반적으로 10~100ms가 적절

```
q = ∞   → FCFS
q = 매우 작음 → 프로세서 공유(Processor Sharing) — 모든 프로세스가 동시에 실행되는 것처럼 보임
```

### 4. 우선순위 스케줄링 (Priority Scheduling)

각 프로세스에 **우선순위**를 부여하고, 가장 높은 우선순위의 프로세스를 먼저 실행.

```
프로세스:   버스트   우선순위 (작은 수 = 높은 우선순위)
P1         10       3
P2          1       1
P3          2       4
P4          1       5
P5          5       2

우선순위 순서: P2(1) → P5(2) → P1(3) → P3(4) → P4(5)

|P2(1)|-- P5(5) --|--- P1(10) ---|P3(2)|P4(1)|
0     1           6            16    18    19

대기 시간: P2=0, P5=1, P1=6, P3=16, P4=18
평균 대기 시간: (0 + 1 + 6 + 16 + 18) / 5 = 8.2ms
```

### 기아 문제와 에이징

**기아(Starvation)**: 우선순위가 낮은 프로세스가 영원히 CPU를 얻지 못하는 현상. 높은 우선순위 프로세스가 계속 들어오면 발생.

**에이징(Aging)**: 기아 문제의 해결책. **시간이 지남에 따라 대기 중인 프로세스의 우선순위를 점진적으로 올려주는 기법**.

```
시간 경과에 따른 에이징 예시:

시간 0:  P_low (우선순위 10, 낮음)
시간 5:  P_low (우선순위 9)    ← 5초 대기했으므로 1 올림
시간 10: P_low (우선순위 8)    ← 10초 대기
...
시간 45: P_low (우선순위 1, 높음)  ← 결국 실행 가능
```

### 멀티레벨 큐 스케줄링

프로세스를 여러 큐로 분류하고, 각 큐에 다른 스케줄링 알고리즘을 적용.

```
┌──────────────────────┐  ← 최고 우선순위
│ 시스템 프로세스 (FCFS)   │
├──────────────────────┤
│ 대화형 프로세스 (RR, q=8) │
├──────────────────────┤
│ 배치 프로세스 (FCFS)      │  ← 최저 우선순위
└──────────────────────┘
```

### 스케줄링 알고리즘 비교표

| 알고리즘 | 방식 | 선점 | 장점 | 단점 |
|---------|------|------|------|------|
| FCFS | 도착 순서 | 비선점 | 단순 | 호위 효과 |
| SJF | 짧은 작업 우선 | 비선점 | 평균 대기 최소 | 실행 시간 예측 어려움 |
| SRTF | 남은 시간 짧은 것 우선 | 선점 | 평균 대기 최소 | 잦은 컨텍스트 스위칭 |
| RR | 시간 할당량 | 선점 | 공정, 응답 빠름 | q 선택이 성능 좌우 |
| 우선순위 | 우선순위 높은 것 | 둘 다 | 중요도 반영 | 기아 문제 |
## 정리

| 항목 | 내용 |
|------|------|
| CPU 스케줄링 | 어떤 프로세스에 CPU를 할당할지 결정 |
| FCFS | 먼저 온 순서대로 처리, 비선점, 호위 효과 |
| SJF | 짧은 작업 우선, 비선점, 이론적 최소 대기 시간 |
| SRTF | SJF의 선점 버전, 남은 시간 짧은 것 우선 |
| Round Robin | 시간 할당량(q)만큼 번갈아 실행, 선점 |
| 우선순위 | 높은 우선순위 먼저, 기아 문제 가능 |
| 에이징 | 대기 시간에 비례하여 우선순위 올려 기아 방지 |
| 반환 시간 | 도착~완료 총 시간 = 대기 + 실행 |
