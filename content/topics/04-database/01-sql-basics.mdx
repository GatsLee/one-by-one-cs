## 핵심 개념

### 관계형 데이터베이스란?

관계형 데이터베이스(RDBMS)는 데이터를 **테이블(표)** 형태로 저장하고 관리하는 시스템이다. 엑셀 스프레드시트를 떠올리면 이해하기 쉽다. 단, 데이터베이스는 수십억 건의 데이터를 빠르게 처리하고, 여러 사용자가 동시에 안전하게 접근할 수 있다.

```
students 테이블:

+----+--------+-----+--------+
| id | name   | age | major  |
+----+--------+-----+--------+
|  1 | Alice  |  22 | CS     |
|  2 | Bob    |  24 | Math   |
|  3 | Charlie|  23 | CS     |
|  4 | Dave   |  21 | Physics|
+----+--------+-----+--------+
 ^        ^
 행(Row)   열(Column)
 = 레코드   = 필드/속성
```

**핵심 용어:**
| 용어 | 엑셀 비유 | 설명 |
|------|----------|------|
| 테이블 (Table) | 시트 | 데이터의 모음. 하나의 주제를 담당 |
| 행 (Row) | 행 | 하나의 데이터 레코드 |
| 열 (Column) | 열 | 데이터의 속성(필드) |
| 기본키 (Primary Key, PK) | - | 각 행을 고유하게 식별하는 열. 중복 불가, NULL 불가 |
| 외래키 (Foreign Key, FK) | - | 다른 테이블의 기본키를 참조하는 열. 테이블 간 관계 설정 |
| 스키마 (Schema) | 시트 구조 | 테이블의 구조 정의 (열 이름, 타입, 제약조건) |

### 키 (Key)

```
students 테이블:
+----+--------+-----+--------+
| id | name   | age | major  |    ← id가 기본키 (Primary Key)
+----+--------+-----+--------+       각 학생을 고유하게 식별
|  1 | Alice  |  22 | CS     |
|  2 | Bob    |  24 | Math   |
+----+--------+-----+--------+

enrollments 테이블:
+----+------------+-----------+-------+
| id | student_id | course_id | grade |
+----+------------+-----------+-------+
|  1 |          1 |       101 | A     |   ← student_id는 외래키 (Foreign Key)
|  2 |          1 |       102 | B+    |     students.id를 참조
|  3 |          2 |       101 | A+    |
+----+------------+-----------+-------+

외래키의 역할:
- student_id = 5 를 넣으려고 하면?
  → students에 id=5인 학생이 없으면 에러! (참조 무결성)
- students에서 id=1인 Alice를 삭제하려면?
  → enrollments에서 참조 중이므로 에러! (또는 연쇄 삭제 설정 가능)
```

### SQL이란?

SQL(Structured Query Language)은 관계형 데이터베이스를 다루는 **표준 언어**다. 데이터를 조회, 삽입, 수정, 삭제하는 데 사용한다.

SQL 명령어는 크게 네 가지로 분류된다:

| 분류 | 이름 | 주요 명령어 | 용도 |
|------|------|-----------|------|
| DDL | Data Definition Language | CREATE, ALTER, DROP | 테이블 구조 정의 |
| DML | Data Manipulation Language | SELECT, INSERT, UPDATE, DELETE | 데이터 조작 |
| DCL | Data Control Language | GRANT, REVOKE | 권한 관리 |
| TCL | Transaction Control Language | COMMIT, ROLLBACK | 트랜잭션 관리 |

이 문서에서는 **DML**과 기본적인 **DDL**을 다룬다.

### 기본 데이터 타입

| 타입 | 설명 | 예시 |
|------|------|------|
| **INTEGER** / **INT** | 정수 | 1, 42, -100 |
| **REAL** / **FLOAT** | 실수 (부동소수점) | 3.14, -2.5 |
| **TEXT** / **VARCHAR(n)** | 문자열. VARCHAR은 최대 길이 지정 | 'hello', 'Alice' |
| **CHAR(n)** | 고정 길이 문자열 | CHAR(10) → 항상 10자 |
| **BOOLEAN** | 참/거짓 | TRUE, FALSE |
| **DATE** | 날짜 | '2026-03-01' |
| **DATETIME** / **TIMESTAMP** | 날짜 + 시간 | '2026-03-01 14:30:00' |
| **BLOB** | 이진 데이터 | 이미지, 파일 등 |

**NULL:** "값이 없음"을 나타내는 특별한 값. 0이나 빈 문자열과 다르다.
- `NULL = NULL` → 결과가 NULL (TRUE가 아님!)
- NULL 비교: `IS NULL`, `IS NOT NULL` 사용

## 코드 예시

### 1. 테이블 생성 (CREATE TABLE)

```sql
-- 학생 테이블 생성
CREATE TABLE students (
    id INTEGER PRIMARY KEY AUTOINCREMENT,  -- 기본키, 자동 증가
    name VARCHAR(50) NOT NULL,             -- 이름, NULL 불가
    age INTEGER CHECK(age >= 0),           -- 나이, 0 이상
    major VARCHAR(50) DEFAULT 'Undeclared', -- 전공, 기본값
    email VARCHAR(100) UNIQUE,             -- 이메일, 중복 불가
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP  -- 생성 시각
);

-- 수업 테이블 생성
CREATE TABLE courses (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(100) NOT NULL,
    credits INTEGER NOT NULL DEFAULT 3,
    professor VARCHAR(50)
);

-- 수강 테이블 생성 (학생과 수업의 관계)
CREATE TABLE enrollments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    student_id INTEGER NOT NULL,
    course_id INTEGER NOT NULL,
    grade VARCHAR(2),
    enrolled_at DATE DEFAULT CURRENT_DATE,
    FOREIGN KEY (student_id) REFERENCES students(id),   -- 외래키
    FOREIGN KEY (course_id) REFERENCES courses(id),      -- 외래키
    UNIQUE(student_id, course_id)  -- 같은 학생이 같은 수업을 중복 수강 불가
);
```

**제약조건 (Constraints):**
| 제약조건 | 설명 |
|---------|------|
| PRIMARY KEY | 기본키. 고유 + NOT NULL |
| NOT NULL | NULL 값 불가 |
| UNIQUE | 중복 값 불가 (NULL은 허용) |
| CHECK | 조건을 만족해야 함 |
| DEFAULT | 값을 지정하지 않으면 기본값 사용 |
| FOREIGN KEY | 다른 테이블의 기본키 참조 |

### 3. 데이터 조회 (SELECT)

```sql
-- === 기본 조회 ===

-- 전체 열 조회 (* = 모든 열)
SELECT * FROM students;

-- 특정 열만 조회
SELECT name, major FROM students;

-- 별칭(Alias) 사용
SELECT name AS 이름, age AS 나이, major AS 전공
FROM students;


-- === WHERE: 조건 필터링 ===

-- CS 전공 학생만
SELECT * FROM students WHERE major = 'CS';

-- 22세 이상인 학생
SELECT * FROM students WHERE age >= 22;

-- CS 전공이고 23세 이상
SELECT * FROM students WHERE major = 'CS' AND age >= 23;

-- CS 또는 Math 전공
SELECT * FROM students WHERE major = 'CS' OR major = 'Math';

-- IN: 여러 값 중 하나와 일치
SELECT * FROM students WHERE major IN ('CS', 'Math');

-- BETWEEN: 범위 조건
SELECT * FROM students WHERE age BETWEEN 22 AND 24;  -- 22, 23, 24 포함

-- LIKE: 패턴 매칭
-- %: 0개 이상의 임의 문자
-- _: 정확히 1개의 임의 문자
SELECT * FROM students WHERE name LIKE 'A%';     -- A로 시작
SELECT * FROM students WHERE name LIKE '%e';     -- e로 끝남
SELECT * FROM students WHERE name LIKE '%li%';   -- li를 포함
SELECT * FROM students WHERE name LIKE '_o%';    -- 두 번째 글자가 o

-- NULL 확인 (= 를 쓰면 안 됨!)
SELECT * FROM students WHERE email IS NULL;
SELECT * FROM students WHERE email IS NOT NULL;

-- NOT: 부정
SELECT * FROM students WHERE major NOT IN ('CS', 'Math');
SELECT * FROM students WHERE NOT (age > 22);
```

### 4. 정렬과 제한 (ORDER BY, LIMIT)

```sql
-- === ORDER BY: 정렬 ===

-- 나이 오름차순 (기본값 ASC)
SELECT * FROM students ORDER BY age;
SELECT * FROM students ORDER BY age ASC;   -- 같은 의미

-- 나이 내림차순
SELECT * FROM students ORDER BY age DESC;

-- 다중 정렬: 전공 오름차순 → 같은 전공 내에서 나이 내림차순
SELECT * FROM students
ORDER BY major ASC, age DESC;


-- === LIMIT: 결과 개수 제한 ===

-- 상위 3개만
SELECT * FROM students ORDER BY age DESC LIMIT 3;

-- OFFSET: 건너뛰기 (페이지네이션에 사용)
-- 2개 건너뛰고 3개 가져오기 (3번째~5번째)
SELECT * FROM students ORDER BY id LIMIT 3 OFFSET 2;

-- 페이지네이션 예시:
-- 1페이지 (1~10번째): LIMIT 10 OFFSET 0
-- 2페이지 (11~20번째): LIMIT 10 OFFSET 10
-- 3페이지 (21~30번째): LIMIT 10 OFFSET 20
```

### 5. 집계 함수와 그룹화 (GROUP BY, HAVING)

```sql
-- === 집계 함수 ===

-- COUNT: 행 개수
SELECT COUNT(*) AS 총학생수 FROM students;

-- 전공별 학생 수
SELECT major, COUNT(*) AS 학생수 FROM students GROUP BY major;

-- SUM: 합계
SELECT SUM(credits) AS 총학점 FROM courses;

-- AVG: 평균
SELECT AVG(age) AS 평균나이 FROM students;

-- MAX, MIN: 최대, 최소
SELECT MAX(age) AS 최고령, MIN(age) AS 최연소 FROM students;

-- 전공별 평균 나이
SELECT major, AVG(age) AS 평균나이, COUNT(*) AS 인원
FROM students
GROUP BY major;

-- 결과:
-- +--------+----------+------+
-- | major  | 평균나이  | 인원  |
-- +--------+----------+------+
-- | CS     | 22.5     | 2    |
-- | Math   | 24.0     | 1    |
-- | Physics| 21.0     | 1    |
-- +--------+----------+------+


-- === HAVING: 그룹 필터 (GROUP BY와 함께 사용) ===

-- WHERE는 그룹화 전에, HAVING은 그룹화 후에 필터링
-- 2명 이상인 전공만
SELECT major, COUNT(*) AS 인원
FROM students
GROUP BY major
HAVING COUNT(*) >= 2;

-- 결과:
-- +--------+------+
-- | major  | 인원  |
-- +--------+------+
-- | CS     | 2    |
-- +--------+------+
```

**SQL 실행 순서:**
```
작성 순서:  SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
실행 순서:  FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT
            1       2         3          4         5         6         7

기억법: "From Where(어디에서) Groups Having Selected data, Order it and Limit"
```

### 6. 데이터 수정과 삭제 (UPDATE, DELETE)

```sql
-- === UPDATE: 데이터 수정 ===

-- Alice의 나이를 23으로 변경
UPDATE students SET age = 23 WHERE name = 'Alice';

-- 여러 열 동시 수정
UPDATE students SET age = 25, major = 'AI' WHERE id = 1;

-- 주의! WHERE 없으면 모든 행이 수정됨!!
-- UPDATE students SET age = 20;  ← 모든 학생의 나이가 20으로!


-- === DELETE: 데이터 삭제 ===

-- 특정 학생 삭제
DELETE FROM students WHERE id = 4;

-- 조건에 맞는 행 삭제
DELETE FROM students WHERE major = 'Physics';

-- 주의! WHERE 없으면 모든 행이 삭제됨!!
-- DELETE FROM students;  ← 모든 학생 데이터 삭제!

-- 안전한 습관: 삭제 전에 SELECT로 먼저 확인
SELECT * FROM students WHERE id = 4;  -- 삭제할 데이터 확인
DELETE FROM students WHERE id = 4;     -- 확인 후 삭제
```

### 7. JOIN: 테이블 결합

JOIN은 두 개 이상의 테이블을 **관련된 열을 기준으로 결합**하는 연산이다.

```sql
-- === INNER JOIN: 양쪽 모두 일치하는 행만 ===
-- "수강 등록한 학생의 이름과 수업명"
SELECT
    s.name AS 학생이름,
    c.name AS 수업명,
    e.grade AS 성적
FROM enrollments e
INNER JOIN students s ON e.student_id = s.id
INNER JOIN courses c ON e.course_id = c.id;

-- 결과:
-- +----------+----------+------+
-- | 학생이름  | 수업명    | 성적  |
-- +----------+----------+------+
-- | Alice    | 데이터베이스| A    |
-- | Alice    | 알고리즘   | B+   |
-- | Bob      | 데이터베이스| A+   |
-- | Bob      | 운영체제   | B    |
-- | Charlie  | 알고리즘   | A    |
-- | Charlie  | 인공지능   | A+   |
-- | Dave     | 데이터베이스| NULL |
-- +----------+----------+------+


-- === LEFT JOIN: 왼쪽 테이블의 모든 행 포함 ===
-- "모든 학생 + 수강 정보 (수강 안 한 학생도 포함)"
SELECT s.name, c.name AS 수업명
FROM students s
LEFT JOIN enrollments e ON s.id = e.student_id
LEFT JOIN courses c ON e.course_id = c.id;

-- LEFT JOIN은 왼쪽 테이블(students)의 모든 행을 포함
-- 매칭되는 오른쪽 데이터가 없으면 NULL로 채움
```

**JOIN 종류 비유:**
```
테이블 A: {1, 2, 3}
테이블 B: {2, 3, 4}

INNER JOIN:  {2, 3}      ← 양쪽 모두 있는 것만
LEFT JOIN:   {1, 2, 3}   ← A의 모든 것 + B에서 매칭되는 것
RIGHT JOIN:  {2, 3, 4}   ← B의 모든 것 + A에서 매칭되는 것
FULL JOIN:   {1, 2, 3, 4} ← 양쪽 모두 (합집합)

시각화:
    A        INNER       LEFT        FULL
 ┌─────┐   ┌─────┐   ┌─────┐    ┌─────┐
 │ ┌───┼───┤     │   │█████│    │█████│
 │ │███│   │ ███ │   │█████│    │█████│
 │ └───┼───┤     │   │█████│    │█████│
 └─────┘   └─────┘   └─────┘    └─────┘
      B      양쪽교집합  A전체+교집합  모두포함
```

### 8. 서브쿼리 (Subquery)

```sql
-- 평균 나이보다 많은 학생 조회
SELECT name, age
FROM students
WHERE age > (SELECT AVG(age) FROM students);

-- 데이터베이스 수업을 듣는 학생 이름
SELECT name FROM students
WHERE id IN (
    SELECT student_id FROM enrollments
    WHERE course_id = (
        SELECT id FROM courses WHERE name = '데이터베이스'
    )
);

-- EXISTS: 서브쿼리에 결과가 있는지 확인
-- 수강 기록이 있는 학생만
SELECT name FROM students s
WHERE EXISTS (
    SELECT 1 FROM enrollments e WHERE e.student_id = s.id
);
```

### 9. 간단한 스키마 설계 예시: 온라인 서점

```sql
-- 사용자 테이블
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 책 테이블
CREATE TABLE books (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL,
    author VARCHAR(100) NOT NULL,
    isbn VARCHAR(13) UNIQUE,
    price INTEGER NOT NULL CHECK(price >= 0),   -- 원 단위
    stock INTEGER NOT NULL DEFAULT 0 CHECK(stock >= 0),
    category VARCHAR(50),
    published_date DATE
);

-- 주문 테이블
CREATE TABLE orders (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    total_amount INTEGER NOT NULL,
    status VARCHAR(20) DEFAULT 'pending',  -- pending, paid, shipped, delivered
    ordered_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 주문 상세 테이블 (다대다 관계 해소)
CREATE TABLE order_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    order_id INTEGER NOT NULL,
    book_id INTEGER NOT NULL,
    quantity INTEGER NOT NULL CHECK(quantity > 0),
    price_at_order INTEGER NOT NULL,  -- 주문 시점의 가격 (가격 변동 대비)
    FOREIGN KEY (order_id) REFERENCES orders(id),
    FOREIGN KEY (book_id) REFERENCES books(id)
);

-- 테이블 관계도:
-- users 1 ──< orders >── order_items >── 1 books
--   한 사용자가 여러 주문을 할 수 있고,
--   한 주문에 여러 책이 포함될 수 있고,
--   한 책이 여러 주문에 포함될 수 있다.

-- 활용 쿼리 예시:

-- 특정 사용자의 주문 내역
SELECT o.id AS 주문번호, b.title AS 책제목, oi.quantity AS 수량,
       oi.price_at_order * oi.quantity AS 소계, o.status AS 상태
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
JOIN books b ON oi.book_id = b.id
WHERE o.user_id = 1
ORDER BY o.ordered_at DESC;

-- 가장 많이 팔린 책 TOP 5
SELECT b.title, SUM(oi.quantity) AS 총판매량
FROM order_items oi
JOIN books b ON oi.book_id = b.id
GROUP BY b.id
ORDER BY 총판매량 DESC
LIMIT 5;

-- 카테고리별 평균 가격
SELECT category, AVG(price) AS 평균가격, COUNT(*) AS 책수
FROM books
GROUP BY category
ORDER BY 평균가격 DESC;
```

### 10. Python에서 SQLite 사용

```python
import sqlite3

# 데이터베이스 연결 (파일이 없으면 새로 생성)
conn = sqlite3.connect('school.db')
cursor = conn.cursor()

# 테이블 생성
cursor.execute('''
    CREATE TABLE IF NOT EXISTS students (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        age INTEGER,
        major TEXT
    )
''')

# 데이터 삽입 (파라미터 바인딩 — SQL 인젝션 방지!)
cursor.execute(
    'INSERT INTO students (name, age, major) VALUES (?, ?, ?)',
    ('Alice', 22, 'CS')
)
```

## 정리

| SQL 명령 | 용도 | 기본 문법 |
|---------|------|----------|
| SELECT | 데이터 조회 | `SELECT 열 FROM 테이블 WHERE 조건` |
| INSERT | 데이터 삽입 | `INSERT INTO 테이블 (열) VALUES (값)` |
| UPDATE | 데이터 수정 | `UPDATE 테이블 SET 열=값 WHERE 조건` |
| DELETE | 데이터 삭제 | `DELETE FROM 테이블 WHERE 조건` |
| CREATE TABLE | 테이블 생성 | `CREATE TABLE 이름 (열 타입 제약조건)` |
| JOIN | 테이블 결합 | `FROM A JOIN B ON A.fk = B.pk` |
| GROUP BY | 그룹화 | `GROUP BY 열` (집계 함수와 함께) |
| ORDER BY | 정렬 | `ORDER BY 열 ASC/DESC` |
| HAVING | 그룹 필터 | `HAVING 집계조건` (GROUP BY 후) |
| LIMIT | 결과 제한 | `LIMIT 개수 OFFSET 시작점` |

**SQL 실행 순서:** FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT

**주의사항:**
- UPDATE/DELETE에 WHERE를 빠뜨리면 모든 행이 영향받는다
- NULL 비교는 `= NULL`이 아닌 `IS NULL` / `IS NOT NULL` 사용
- 사용자 입력을 SQL에 직접 넣지 말 것 → 파라미터 바인딩(?) 사용 (SQL 인젝션 방지)
